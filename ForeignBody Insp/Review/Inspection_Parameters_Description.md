# 검사 파라미터 상세 설명서 (Inspection Parameters Description)

본 문서는 바이알 이물 검사 시스템의 **Inspection Parameters** 설정창에 포함된 모든 파라미터의 **단위, 범위, 기본값, 동작 원리, 상호 연동 관계**를 정확하게 기술합니다.

> **용어 규칙**
> - **px** = 이미지 픽셀 (해상도에 따라 실제 크기가 달라짐)
> - **px²** = 픽셀 면적 (윤곽선 내부 픽셀 수)
> - **gray level** = 0~255 범위의 8비트 밝기 값
> - **비율** = 0.0~1.0 범위의 무차원 계수
> - **프레임** = 카메라/동영상의 한 장 단위

---

## 1. 공통 제어 및 디버깅 뷰 (Common Controls & View)

### 1.1 검사 모드 선택 (Inspection Mode)
| 모드                 | 설명                                                                      |
| ------------------ | ----------------------------------------------------------------------- |
| **기존 검사 (Legacy)** | 단일 프레임 Threshold + 형태학 기반. 정지된 이미지에서 밝기 차이로 이물을 검출한다.                   |
| **수정 검사 (ECC)**    | 연속 프레임 정렬(Alignment) + 차영상(AbsDiff) 기반. 기준 프레임 대비 변화만 추출하여 미세 이물을 검출한다. |

두 모드는 **정지 인식(State Gate)** 로직을 공유하며, 정지가 확인된 프레임에서만 검사를 수행한다.

> **cf. 용어 해설**
> - **Threshold**: 이진화를 위한 밝기 기준값. 이 값을 기준으로 픽셀을 전경/배경으로 분류한다.
> - **Morphology (형태학)**: 이진화 영상에 구조적 연산(침식, 팽창, 열림, 닫힘)을 적용하여 노이즈 제거나 형태 보정을 수행하는 영상처리 기법.
> - **ECC (Enhanced Correlation Coefficient)**: 두 영상 간 상관계수를 최대화하는 반복적 정렬 알고리즘. 밝기 기반으로 서브픽셀 정밀도의 정렬을 수행한다.
> - **AbsDiff (Absolute Difference)**: 두 영상의 각 픽셀 밝기 차이의 절대값을 계산하는 연산. 차영상 생성에 사용된다.
> - **Alignment (정렬)**: 기준 영상과 현재 영상의 위치/회전을 맞추는 과정. 차영상 비교 전 필수적인 전처리 단계이다.
> - **State Gate (상태 게이트)**: 바이알의 운동 상태(운행/정지)를 판별하는 상태 머신 로직. 정지 확정 시에만 검사를 수행한다.
> - **프레임**: 카메라 또는 동영상에서 한 번에 캡처되는 단일 이미지 단위.

### 1.2 실시간 디버깅 제어
| 항목 | 설명 |
|------|------|
| **실시간 디버깅** | 체크 시 파라미터 변경 즉시 백그라운드에서 검사를 수행하고 결과를 뷰에 업데이트한다. |
| **파라미터별 View 표시** | 체크 시 특정 파라미터 조절 시 해당 연산 단계의 중간 이미지로 자동 전환된다. |
| **표시 (Debug View)** | 원본 Gray ~ 최종 결과(Result), 중간 단계(Opening, Closing, Diff Map 등) 총 16가지 뷰 선택 가능. |
| **맞춤 버튼** | 이미지를 현재 뷰 크기에 맞게 배율 조정한다. |
| **상태바 (Footer)** | 마우스 커서 위치의 픽셀 좌표(X, Y)와 밝기값(0~255)을 실시간 표시한다. |

> **디바운스(Debounce)**: 스핀박스를 빠르게 클릭해도 마지막 입력 후 약 200ms 대기 후에 검사가 시작되므로 UI가 멈추지 않는다.

> **cf. 용어 해설**
> - **디바운스 (Debounce)**: 연속적인 입력 이벤트가 발생할 때 마지막 이벤트 이후 일정 시간이 지난 뒤에만 실제 동작을 실행하는 기법. UI 응답성을 유지하기 위해 사용한다.
> - **Debug View (디버그 뷰)**: 영상처리 파이프라인의 중간 단계 결과를 시각적으로 확인할 수 있는 표시 모드.
> - **Gray (그레이스케일)**: 컬러 영상을 밝기 정보만 남긴 단일 채널 영상으로 변환한 것. 0(검정)~255(흰색) 범위.
> - **Opening**: 모폴로지 열림 연산(침식 후 팽창). 작은 돌출부나 점 노이즈를 제거한다.
> - **Closing**: 모폴로지 닫힘 연산(팽창 후 침식). 작은 구멍이나 끊어진 부분을 메운다.
> - **Diff Map (차영상 맵)**: 두 영상 간 픽셀별 차이를 시각화한 영상.

## 2. 탭 1: 분류/검출 조건 (Classification/Detection)

이 탭은 **기존 검사(Legacy) 모드**의 일반 이물 검출 파라미터와, 양 모드 공유인 **상태 게이트**를 설정한다.

### 2.1 상태 게이트 (State Gate) — 정지 인식

바이알이 정지했는지 판단하는 상태 머신이다. **기존 검사 / ECC 검사 모두** 이 로직을 공유한다.

#### 동작 원리

매 프레임마다 3가지 변화량을 계산한다:

| 변화량 | 계산 방법 | 단위 | 의미 |
|--------|-----------|------|------|
| **Motion Score** | 인접 프레임 간 ROI 영역의 MSE (Mean Squared Error) | gray level² | 전체 밝기 변화 크기. 바이알 이동 시 급격히 상승한다. |
| **Edge Change** | 인접 프레임 간 Canny 에지맵의 absdiff 평균 | gray level (0~255 스케일) | 에지(윤곽) 변화량. 바이알 테두리 진입/이탈에 민감하다. |
| **Dark Change** | 인접 프레임 간 어두운 영역(outer_mask_threshold 이하) 면적 비율의 절대 차 | 비율 (0.0~1.0) | 바이알 본체가 ROI에 들어오거나 나갈 때 크게 변한다. |

#### 상태 전이 규칙

```
running → settling → stop_candidate → stop_confirmed
```

| 조건 | 판정 | 결과 |
|------|------|------|
| Motion ≥ Reenter Motion **OR** Edge ≥ Reenter Edge **OR** Dark ≥ Reenter Dark | **운행(Running)** | 안정 카운트 리셋, 검사 스킵 |
| Motion ≤ Stop Motion **AND** Edge ≤ Stop Edge **AND** Dark ≤ Stop Dark | **안정(Stable)** | 안정 카운트 +1 |
| 안정 카운트 ≥ Confirm Frames | **정지 확정(stop_confirmed)** | 실제 검사 수행 |
| Candidate Frames ≤ 안정 카운트 < Confirm Frames | **정지 후보(stop_candidate)** | 추가 확인 중 |
| 위 어느 것도 아님 | **안정화 중(settling)** | 안정 카운트 리셋 |

#### 파라미터 상세

| 파라미터 | 단위 | 범위 | 기본값 (Legacy/ECC) | 의미 |
|----------|------|------|---------------------|------|
| **정지 motion 상한** | gray level² | 0.0~255.0 | 3.0 / 5.0 | 이 값 **이하**여야 "안정"으로 누적. 낮을수록 엄격한 정지 판정. |
| **정지 edge 상한** | gray level | 0.0~255.0 | 4.0 / 6.0 | Canny 에지 변화가 이 값 이하여야 안정. |
| **정지 dark 변화** | 비율 | 0.000~1.000 | 0.02 / 0.02 | 어두운 영역 비율 변화가 이 값 이하여야 안정. 0.02 = 2% 이내. |
| **재진입 motion 상한** | gray level² | 0.0~255.0 | 12.0 / 12.0 | 이 값 **이상**이면 즉시 running으로 복귀. Stop 상한보다 높아야 한다. |
| **재진입 edge 상한** | gray level | 0.0~255.0 | 12.0 / 12.0 | 에지 변화가 다시 커지면 running 복귀. |
| **재진입 dark 변화** | 비율 | 0.000~1.000 | 0.05 / 0.05 | dark 비율 변화가 다시 커지면 running 복귀. |
| **후보 연속 프레임** | 프레임 | 1~100 | 2 / 2 | 안정 상태가 이 횟수만큼 연속되면 stop_candidate. |
| **확정 연속 프레임** | 프레임 | 1~100 | 4 / 4 | 안정 상태가 이 횟수만큼 유지되면 stop_confirmed(검사 시작). |

> **연동 관계**: `Stop 상한 < Reenter 상한` 이어야 히스테리시스가 동작한다. 두 값이 같으면 채터링(상태 진동)이 발생할 수 있다.
> `Candidate Frames < Confirm Frames`이어야 하며, Confirm Frames가 클수록 정지 확정까지 시간이 오래 걸리지만 오판이 줄어든다.

> **cf. 용어 해설**
> - **MSE (Mean Squared Error, 평균 제곱 오차)**: 두 영상의 대응 픽셀 밝기 차이를 제곱하여 평균한 값. 영상 간 전체적인 변화량을 하나의 수치로 나타낸다.
> - **Canny**: 다단계 에지 검출 알고리즘. 가우시안 블러 → 그래디언트 계산 → 비최대 억제 → 히스테리시스 임계값의 과정으로 선명한 에지를 추출한다.
> - **absdiff (Absolute Difference)**: 두 영상의 픽셀별 밝기 차이의 절대값. `|img1 - img2|`를 계산한다.
> - **히스테리시스 (Hysteresis)**: 상태 전이에 서로 다른 상한/하한 임계값을 적용하여, 경계 근처에서 상태가 빈번하게 전환되는 것을 방지하는 기법.
> - **상태 머신 (State Machine)**: 유한한 상태들과 상태 간 전이 규칙으로 시스템의 동작을 모델링하는 구조. 여기서는 running → settling → stop_candidate → stop_confirmed 의 상태를 관리한다.
> - **ROI (Region of Interest, 관심 영역)**: 전체 영상 중 실제 검사 대상이 되는 부분 영역. 불필요한 영역을 제외하여 연산량을 줄이고 오검출을 방지한다.
> - **채터링 (Chattering)**: 임계값 경계 근처에서 상태가 빈번하게 전환(진동)되는 현상. 히스테리시스를 적용하여 방지한다.
> - **Motion Score**: 인접 프레임 간 ROI 영역의 MSE로 계산되는 전체 밝기 변화량 지표.
> - **Edge Change**: 인접 프레임 간 Canny 에지맵의 absdiff 평균으로 측정되는 윤곽 변화량.
> - **Dark Change**: 인접 프레임 간 어두운 영역 면적 비율의 절대 차이. 바이알 본체의 진입/이탈을 감지한다.
> - **gray level (밝기 레벨)**: 8비트 그레이스케일 영상에서 각 픽셀의 밝기를 나타내는 0~255 범위의 정수값.
> - **프레임**: 카메라 또는 동영상에서 한 번에 캡처되는 단일 이미지. 연속 프레임 간 비교로 변화를 감지한다.
> - **Reenter (재진입)**: 정지 상태에서 다시 운행 상태로 복귀하는 전이. Reenter 임계값을 초과하면 즉시 running 상태로 돌아간다.
> - **Stop (정지)**: 변화량이 Stop 임계값 이하일 때 안정 상태로 판정하는 기준.

### 2.2 일반 이물 검출 파라미터

기존 검사(Legacy) 모드에서 Gray → Blur → Threshold → Morphology → Contour 순서로 사용된다.

| 파라미터 | 단위 | 범위 | 기본값 | 동작 및 연동 |
|----------|------|------|--------|-------------|
| **Threshold** | gray level | 0~255 | 100 (코드) / 220 (현재 설정) | 이진화 기준 밝기. 이 값보다 **어두운** 픽셀이 이물 후보가 된다. `Adaptive Threshold` 사용 시 보조 역할(global dark mask)을 한다. |
| **Adaptive Threshold** | on/off | — | on | 체크 시 `cv2.adaptiveThreshold(blockSize=15, C=3)` + 위 Threshold의 global dark mask를 AND 결합한다. 조명이 불균일한 바이알 곡면에 효과적. |
| **Min Area** | px² | 0~5,000 | 10 | 윤곽선 면적이 이 값 미만이면 노이즈로 제거. 값을 높이면 작은 이물을 놓칠 수 있다. |
| **Max Area** | px² | 1~100,000 | 5,000 (코드) / 500 (현재 설정) | 윤곽선 면적이 이 값 초과면 구조물로 제거. 바이알 전체 영역이 잡히는 것을 방지. |
| **Min Circularity** | 비율 | 0.00~1.00 | 0.0 | `4π × Area / Perimeter²`. 0이면 제한 없음. 1.0이면 완전 원만 통과. 길쭉한 스크래치를 걸러낼 때 0.3~0.5 사용. |
| **Min Solidity** | 비율 | 0.00~1.00 | 0.0 | `Area / ConvexHullArea`. 윤곽선 내부에 빈 공간이 많으면 낮아진다. 불규칙 노이즈 필터링용. |
| **Max Aspect Ratio** | 무차원 | 1.0~50.0 | 10.0 | `max(w,h) / min(w,h)`. 바운딩박스 장축/단축 비. 값이 클수록 길쭉한 객체도 허용한다. |
| **Open 커널** | px (커널 한 변 크기) | 1~15 | 2 | 모폴로지 Opening(침식→팽창) 커널. 이진화 후 작은 점 노이즈를 제거한다. 값이 클수록 더 큰 노이즈까지 제거하지만, 작은 실제 이물도 사라질 수 있다. |
| **Close 커널** | px (커널 한 변 크기) | 1~15 | 3 | 모폴로지 Closing(팽창→침식) 커널. 끊어진 이물 조각을 연결하고 내부 구멍을 메운다. 값이 클수록 이물이 합쳐지지만 가까운 별개 이물도 붙을 수 있다. |

> **처리 흐름**: `Gray → GaussianBlur(3,3) → [Adaptive/Static] Threshold → Opening(open_kernel) → Closing(close_kernel) → findContours → 면적/형상 필터`
>
> **연동 관계**: `Threshold`↔`Adaptive Threshold` — Adaptive가 켜지면 Threshold는 global dark mask 역할만 한다. `Min/Max Area`는 이후 형상 필터(Circularity, Solidity, Aspect)와 **AND 조건**으로 모두 통과해야 검출된다.

> **cf. 용어 해설**
> - **Threshold (임계값)**: 영상 이진화의 기준이 되는 밝기값. 이 값을 기준으로 픽셀을 흰색(전경) 또는 검정(배경)으로 분류한다.
> - **이진화 (Binarization)**: 그레이스케일 영상의 각 픽셀을 임계값 기준으로 0 또는 255 두 값으로 변환하는 과정.
> - **Adaptive Threshold (적응 임계값)**: 영상의 지역별 밝기 분포를 고려하여 각 픽셀마다 다른 임계값을 적용하는 이진화 방식. 조명이 불균일한 환경에서 효과적이다.
> - **GaussianBlur (가우시안 블러)**: 가우시안(정규분포) 함수를 커널로 사용하는 평활화 필터. 영상의 고주파 노이즈를 부드럽게 제거한다.
> - **Morphology (형태학 연산)**: 구조 요소(커널)를 이용하여 이진 영상의 형태를 변형하는 연산 집합.
> - **Opening (열림)**: 침식(Erosion) 후 팽창(Dilation)을 순서대로 적용하는 모폴로지 연산. 작은 돌출부와 점 노이즈를 제거한다.
> - **Closing (닫힘)**: 팽창(Dilation) 후 침식(Erosion)을 순서대로 적용하는 모폴로지 연산. 작은 구멍과 끊어진 부분을 메운다.
> - **침식 (Erosion)**: 커널 영역 내 최솟값을 취하여 밝은 영역을 줄이는 모폴로지 기본 연산. 작은 밝은 점이 제거된다.
> - **팽창 (Dilation)**: 커널 영역 내 최댓값을 취하여 밝은 영역을 넓히는 모폴로지 기본 연산. 끊어진 영역이 연결된다.
> - **findContours**: OpenCV에서 이진 영상의 외곽선(경계)을 추적하여 윤곽선 좌표 목록을 반환하는 함수.
> - **Contour (윤곽선)**: 이진 영상에서 동일한 밝기 값을 가지는 연속된 점들의 경계선. 객체의 형태를 수치적으로 표현한다.
> - **Circularity (원형도)**: `4π × Area / Perimeter²`로 계산되는 형상 지표. 완전한 원은 1.0, 길쭉하거나 불규칙할수록 0에 가깝다.
> - **Solidity (충실도)**: `Area / ConvexHullArea`로 계산되는 형상 지표. 윤곽선이 볼록 껍질에 얼마나 꽉 차있는지를 나타낸다.
> - **Convex Hull (볼록 껍질)**: 윤곽선의 모든 점을 포함하는 가장 작은 볼록 다각형. 윤곽선의 오목한 부분을 무시한 외곽 형태이다.
> - **Aspect Ratio (종횡비)**: 바운딩박스의 장축 길이를 단축 길이로 나눈 값. 1에 가까울수록 정사각형/원형, 클수록 길쭉한 형태.
> - **px² (픽셀 면적)**: 윤곽선 내부에 포함된 픽셀의 총 개수. 객체의 크기를 나타내는 단위.
> - **윤곽선**: 이진 영상에서 객체의 경계를 이루는 연속된 픽셀 좌표의 집합.
> - **커널 (Kernel)**: 영상처리에서 필터링이나 모폴로지 연산에 사용되는 작은 행렬. 연산의 영향 범위와 형태를 결정한다.
> - **gray level (밝기 레벨)**: 8비트 그레이스케일 영상에서 각 픽셀의 밝기를 나타내는 0~255 범위의 정수값.

### 2.3 분류 조건

| 파라미터 | 단위 | 범위 | 기본값 | 의미 |
|----------|------|------|--------|------|
| **Noise/Particle 대비 임계값** | gray level | 0~255 | 10 (현재 설정) / 30 (코드 기본) | 검출된 이물 영역의 평균 밝기 대비(Contrast)가 이 값 이상이면 **Particle**, 미만이면 **Noise_Dust**로 분류한다. 룰 기반 분류에서만 사용된다. |
| **디버그 필터** | 선택 | ALL / NOISE / PARTICLE | ALL | 디버그 뷰에 표시할 이물 종류를 선택한다. 검출 자체에는 영향 없다. |

> **cf. 용어 해설**
> - **Contrast (대비)**: 검출된 이물 영역의 평균 밝기와 주변 배경 밝기 간의 차이. 이물이 배경 대비 얼마나 두드러지는지를 나타낸다.
> - **Noise_Dust (노이즈/먼지)**: 대비가 낮은 검출 객체에 부여되는 분류 라벨. 실제 이물보다는 미세 먼지나 센서 노이즈일 가능성이 높은 후보.
> - **Particle (입자)**: 대비가 높은 검출 객체에 부여되는 분류 라벨. 실제 이물(이물 입자)일 가능성이 높은 후보.
> - **룰 기반 분류**: 학습된 모델이 아닌 사전 정의된 규칙(임계값, 조건문)으로 객체를 분류하는 방식. 여기서는 대비 임계값으로 Noise_Dust/Particle을 구분한다.

## 3. 탭 2: Bubble 검출 (Bubble Detection)

DoG(Difference of Gaussians) 알고리즘으로 액체 내 기포를 특화 검출한다. 기존 검사(Legacy)에서만 사용되며, 일반 검출과 **병렬로 동시 실행** 후 결과를 병합한다.

### 3.1 처리 파이프라인

```
배경 평탄화(Morph Opening) → 양극성 차이 → [CLAHE] → 노이즈 제거 → DoG 밴드패스 → MAD 적응 임계 → 형태학 정리 → 형상 필터
```

> **cf. 용어 해설**
> - **DoG (Difference of Gaussians, 가우시안 차이)**: 서로 다른 σ 값의 두 가우시안 블러 결과를 빼서 특정 크기 범위의 구조만 강조하는 밴드패스 필터 기법.
> - **MAD (Median Absolute Deviation, 중앙값 절대 편차)**: 데이터의 중앙값으로부터 각 값의 편차의 중앙값. 이상치에 강건한 산포 측정 방법으로, 적응 임계값 산출에 사용된다.
> - **밴드패스 (Bandpass)**: 특정 주파수(크기) 범위의 신호만 통과시키고 나머지는 억제하는 필터. DoG가 공간 주파수 밴드패스 역할을 한다.
> - **파이프라인 (Pipeline)**: 여러 영상처리 단계가 순차적으로 연결된 처리 흐름. 각 단계의 출력이 다음 단계의 입력이 된다.
> - **CLAHE (Contrast Limited Adaptive Histogram Equalization)**: 지역적 히스토그램 평활화에 대비 제한(클리핑)을 적용한 영상 대비 향상 기법.
> - **Morph Opening (모폴로지 열림)**: 침식 후 팽창을 적용하여 배경의 큰 구조만 추출하는 연산. 여기서는 배경 평탄화에 사용된다.

### 3.2 배경 평탄화

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **배경 커널 (bg_open_ksize)** | px (홀수 강제) | 3~999 | 61 / 20 | Morphological Opening으로 바이알 곡면의 큰 배경 구조만 추출하는 커널 크기. 바이알 직경/곡률에 맞춰 설정한다. 큰 값일수록 더 넓은 배경을 추정한다. |
| **배경 스무딩 σ (bg_smooth_sigma)** | px (σ값) | 0.0~ | 15.0 / 2.0 | 배경 추정 후 Gaussian 블러의 표준편차. 0이면 자동으로 `ksize/4` 적용. 값이 클수록 배경이 더 매끄러워진다. |

> **동작**: 원본에서 배경을 빼면(`flat = max(bg-work, work-bg)`) 배경 밝기 편차가 제거되어 기포 신호만 남는다.

> **cf. 용어 해설**
> - **Morphological Opening (모폴로지 열림)**: 큰 커널로 침식 후 팽창을 수행하여 작은 구조를 제거하고 넓은 배경 성분만 추출하는 연산. 배경 추정에 활용된다.
> - **배경 평탄화 (Background Flattening)**: 영상에서 조명 불균일이나 곡면에 의한 밝기 편차를 제거하는 전처리. 원본에서 추정된 배경을 빼서 균일한 밝기 분포를 만든다.
> - **σ (Sigma, 시그마)**: 가우시안 함수의 표준편차. 블러의 강도(확산 범위)를 결정하는 파라미터. 값이 클수록 더 넓은 범위를 평활화한다.
> - **GaussianBlur (가우시안 블러)**: 가우시안(정규분포) 함수를 커널로 사용하는 평활화 필터. 영상의 고주파 성분을 부드럽게 제거한다.
> - **커널 (Kernel)**: 영상처리에서 필터링이나 모폴로지 연산에 사용되는 작은 행렬. 여기서는 배경 추정의 영향 범위를 결정한다.

### 3.3 CLAHE (선택적 대비 보정)

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **CLAHE 사용** | on/off | — | off | 평탄화된 영상에 CLAHE 대비 보정을 추가 적용한다. |
| **Clip Limit** | 무차원 | 0.1~ | 2.0 / 20.0 | 히스토그램 클리핑 한계. 높을수록 대비 증가가 강하다. |
| **Grid Size** | px (타일 한 변) | 1~ | 8 / 64 | CLAHE 타일 격자 크기. 작을수록 지역적 보정이 세밀하지만 노이즈 증폭 위험. |

> **cf. 용어 해설**
> - **CLAHE (Contrast Limited Adaptive Histogram Equalization)**: 영상을 작은 타일로 나누어 각 타일별 히스토그램 평활화를 수행하되, 대비 증폭을 제한(클리핑)하여 노이즈 증폭을 억제하는 기법.
> - **히스토그램 (Histogram)**: 영상의 밝기값별 픽셀 빈도를 나타내는 분포도. 히스토그램 평활화는 이 분포를 균일하게 만들어 대비를 향상시킨다.
> - **클리핑 (Clipping)**: CLAHE에서 히스토그램의 특정 빈(bin)이 한계값을 초과하면 잘라내어 다른 빈에 재분배하는 과정. 과도한 대비 증가를 방지한다.
> - **타일 (Tile)**: CLAHE에서 영상을 분할하는 격자 단위. 각 타일 내에서 독립적으로 히스토그램 평활화가 수행된다. 크기가 작을수록 지역 보정이 세밀하다.

### 3.4 노이즈 제거

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **Denoise Mode** | 선택 | median / bilateral / none | median / none | DoG 연산 전 미세 잡음을 억제하는 필터 종류. |
| **Median 커널** | px (홀수) | 3~ | 5 / 3 | Median 블러 커널 크기. 작은 솔트&페퍼 노이즈 제거에 효과적. |
| **Bilateral d** | px | 1~ | 7 | Bilateral 필터의 이웃 직경. |
| **Bilateral σColor** | gray level | 0.0~ | 25.0 | 색상 공간에서의 필터 σ. 값이 크면 밝기 차가 큰 픽셀도 혼합된다. |
| **Bilateral σSpace** | px | 0.0~ | 25.0 | 좌표 공간에서의 필터 σ. 값이 크면 먼 픽셀까지 영향을 미친다. |

> **cf. 용어 해설**
> - **Median Filter (중앙값 필터)**: 커널 내 픽셀값을 정렬하여 중앙값을 출력으로 사용하는 비선형 필터. 에지를 보존하면서 점 형태의 노이즈를 효과적으로 제거한다.
> - **Bilateral Filter (양측 필터)**: 공간적 거리와 밝기 차이를 동시에 고려하는 에지 보존 평활화 필터. 에지 경계는 유지하면서 평탄한 영역의 노이즈만 제거한다.
> - **Salt & Pepper Noise (소금&후추 노이즈)**: 영상에 무작위로 나타나는 흰색(255) 또는 검정(0) 점 형태의 잡음. 센서 결함이나 전송 오류로 발생한다.
> - **σ (Sigma, 시그마)**: 필터의 확산 범위를 제어하는 표준편차 파라미터. σColor는 밝기 공간, σSpace는 좌표 공간에서의 영향 범위를 결정한다.

### 3.5 DoG 밴드패스

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **σ small** | px (σ값) | 0.1~ | 1.2 / 5.0 | 작은 Gaussian 블러의 σ. 검출할 기포의 **최소 두께**에 대응. |
| **σ large** | px (σ값) | σ small 이상 | 6.0 / 20.0 | 큰 Gaussian 블러의 σ. 검출할 기포의 **최대 두께**에 대응. |

> **동작**: `DoG = |GaussianBlur(σ_small) - GaussianBlur(σ_large)|`
> σ_small~σ_large 사이 크기의 구조만 강조된다. 기포보다 작은 노이즈와 기포보다 큰 배경 구조가 동시에 억제된다.
> **연동**: σ_small이 작을수록 미세 기포까지 잡지만 노이즈도 증가. σ_large가 클수록 큰 기포까지 커버하지만 배경 간섭이 늘어난다.

> **cf. 용어 해설**
> - **DoG (Difference of Gaussians, 가우시안 차이)**: 서로 다른 σ 값의 두 가우시안 블러 결과를 빼서 특정 크기 범위의 구조만 강조하는 밴드패스 필터. `|Blur(σ_small) - Blur(σ_large)|`로 계산된다.
> - **Gaussian (가우시안)**: 정규분포(종형 곡선) 함수. 영상처리에서 블러 커널의 가중치 분포로 사용되며, σ가 클수록 넓게 평활화된다.
> - **σ (Sigma, 시그마)**: 가우시안 함수의 표준편차. 블러의 확산 범위를 결정하며, 검출 대상의 공간적 크기와 직결된다.
> - **밴드패스 필터 (Bandpass Filter)**: 특정 공간 주파수(크기) 범위의 구조만 통과시키고, 그보다 작은(고주파) 노이즈와 큰(저주파) 배경을 동시에 억제하는 필터.
> - **노이즈 (Noise)**: 영상에서 실제 신호가 아닌 불필요한 잡음 성분. 센서 특성, 전자적 간섭, 조명 변화 등에 의해 발생한다.

### 3.6 MAD 적응 임계

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **thr_k (MAD 계수)** | 무차원 (배수) | 0.5~20.0 | 4.0 / 17.5 | 기포 후보 판정 민감도. `임계값 = median + k × 1.4826 × MAD`. |

> **동작**: MAD(Median Absolute Deviation) 기반 통계적 이진화. DoG 결과에서 중앙값 + k×MAD 이상인 픽셀만 기포 후보가 된다.
> **낮은 k** = 민감(더 많은 후보, 오검출 증가), **높은 k** = 둔감(확실한 기포만 검출).
> 1.4826은 정규분포 가정 시 MAD→σ 변환 계수이다.

> **cf. 용어 해설**
> - **MAD (Median Absolute Deviation, 중앙값 절대 편차)**: 데이터에서 중앙값을 구한 뒤, 각 값과 중앙값의 차이(절대값)의 중앙값. `MAD = median(|Xi - median(X)|)`. 이상치에 강건한 산포 측정 방법이다.
> - **Median (중앙값)**: 데이터를 크기순으로 정렬했을 때 정중앙에 위치하는 값. 평균과 달리 극단값(이상치)의 영향을 받지 않는다.
> - **적응 임계 (Adaptive Threshold)**: 영상의 통계적 특성(여기서는 MAD)에 기반하여 임계값을 자동으로 결정하는 방식. 고정 임계값보다 다양한 조건에서 안정적으로 동작한다.
> - **정규분포 (Normal Distribution)**: 평균을 중심으로 좌우 대칭인 종형 확률분포. 자연 현상의 많은 데이터가 이 분포를 따른다.
> - **1.4826 변환 계수**: 정규분포를 가정할 때 `σ ≈ 1.4826 × MAD`가 성립하는 수학적 상수. MAD를 표준편차(σ)와 동등한 스케일로 변환하기 위해 사용된다.

### 3.7 형태학 정리

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **Morph Close** | px (홀수 커널) | 1~31 | 5 | 기포 후보의 내부 구멍을 메운다. |
| **Morph Open** | px (홀수 커널) | 1~31 | 3 / 5 | 작은 점 노이즈를 제거한다. |

> **cf. 용어 해설**
> - **Morphology (형태학 연산)**: 구조 요소(커널)를 이용하여 이진 영상의 형태를 변형하는 연산 집합. 노이즈 제거, 형태 보정, 객체 분리/연결 등에 사용된다.
> - **Opening (열림)**: 침식 후 팽창을 적용하는 모폴로지 연산. 작은 돌출부와 점 노이즈를 제거하면서 큰 객체의 전체적 크기는 유지한다.
> - **Closing (닫힘)**: 팽창 후 침식을 적용하는 모폴로지 연산. 작은 구멍과 끊어진 부분을 메우면서 큰 객체의 전체적 크기는 유지한다.
> - **커널 (Kernel)**: 모폴로지 연산에 사용되는 구조 요소. 커널의 크기가 클수록 연산의 영향 범위가 넓어진다. 홀수 크기를 사용하여 중심점을 명확히 한다.

### 3.8 형상 필터

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **Min Diameter** | px | 1~500 | 8 / 10 | 최소 기포 직경. `minEnclosingCircle` 반지름이 이 값/2 미만이면 제거. |
| **Max Diameter** | px | 1~1,000 | 100 / 150 | 최대 기포 직경. 이보다 큰 원형 객체는 기포가 아닌 것으로 판단. |
| **Circularity Min** | 비율 | 0.00~1.00 | 0.35 / 0.4 | `4π × Area / Perimeter²`. 기포는 둥글므로 0.35~0.5 이상을 권장. |
| **Solidity Min** | 비율 | 0.00~1.00 | 0.70 | `Area / ConvexHullArea`. 기포는 꽉 찬 형태이므로 0.7 이상 권장. |
| **Max Aspect Ratio** | 무차원 | 1.0~10.0 | 2.2 | 장축/단축 비. 기포는 거의 원형이므로 2.0~2.5가 적합. |

> **면적 자동 계산**: Diameter로부터 내부적으로 면적 범위가 자동 계산된다.
> `min_area = π × (min_diameter/2)² × 0.35`, `max_area = π × (max_diameter/2)² × 1.30`
> 즉 **Diameter, Circularity, Solidity, Aspect Ratio 모두 AND 조건**으로 전부 통과해야 기포로 확정된다.

> **cf. 용어 해설**
> - **minEnclosingCircle (최소 외접원)**: 윤곽선의 모든 점을 포함하는 가장 작은 원. 반지름과 중심 좌표를 반환하며, 기포 직경 측정에 사용된다.
> - **Circularity (원형도)**: `4π × Area / Perimeter²`로 계산되는 형상 지표. 완전한 원은 1.0, 불규칙하거나 길쭉할수록 0에 가깝다.
> - **Solidity (충실도)**: `Area / ConvexHullArea`로 계산되는 형상 지표. 1.0에 가까울수록 윤곽선이 볼록 껍질에 꽉 차있음을 의미한다.
> - **Convex Hull (볼록 껍질)**: 윤곽선의 모든 점을 포함하는 가장 작은 볼록 다각형. 윤곽선의 오목한 부분을 무시한 외곽 형태이다.
> - **Aspect Ratio (종횡비)**: 바운딩박스의 장축 길이를 단축 길이로 나눈 비율. 기포는 원형에 가까우므로 1에 가까운 값을 가진다.
> - **Diameter (직경)**: 최소 외접원의 지름(2 × 반지름). 기포의 크기를 나타내는 대표 수치.
> - **Perimeter (둘레)**: 윤곽선의 전체 길이. 원형도 계산에 사용되며, `cv2.arcLength`로 측정한다.
> - **π (파이)**: 원주율(약 3.14159). 원형도 공식 및 면적 자동 계산에 사용되는 수학 상수.

## 4. 탭 3: ECC 정렬 검사 (ECC Alignment Inspection)

기준 프레임(Reference)과 현재 프레임을 픽셀 단위로 정렬한 후, 차영상에서 이물을 검출한다.

### 4.1 전체 파이프라인 요약

```
프레임 입력 → 상태 게이트 판정 → [stop_confirmed일 때만]
→ 기준 프레임과 정렬(Alignment) → 유효 마스크 생성 → Safe Liquid ROI 적용
→ 차영상 계산 → Sweet Spot 판정 → 이진화 → Blob 필터 → [트래킹] → 최종 결과
```

> **cf. 용어 해설**
> - **ECC (Enhanced Correlation Coefficient)**: 두 영상 간 상관계수를 최대화하는 반복적 정렬 알고리즘. 밝기 기반으로 서브픽셀 정밀도의 기하학적 변환을 추정한다.
> - **Reference (기준 프레임)**: 정렬 및 차영상 비교의 기준이 되는 영상. 이물이 없는 정상 상태의 프레임을 저장하여 사용한다.
> - **Alignment (정렬)**: 기준 영상과 현재 영상의 기하학적 위치(이동, 회전)를 맞추는 과정.
> - **Blob**: 이진 영상에서 연결된 흰색 픽셀의 덩어리. 이물 후보 영역을 나타낸다.
> - **트래킹 (Tracking)**: 여러 프레임에 걸쳐 동일한 후보를 추적하여 일시적 노이즈와 실제 이물을 구별하는 과정.
> - **차영상**: 기준 영상과 현재 영상의 픽셀별 차이를 계산한 영상. 변화가 발생한 영역이 밝게 나타난다.
> - **Sweet Spot**: 차영상의 평균 변화량이 적절한 범위(상한~하한) 내에 있어 검사에 적합한 상태.
> - **파이프라인 (Pipeline)**: 전처리부터 최종 결과까지 순차적으로 연결된 영상처리 흐름.

### 4.2 기준 템플릿 관리

| 기능 | 설명 |
|------|------|
| **현재 화면 저장** | 현재 화면과 검사 ROI를 ECC 기준(Reference)으로 영구 저장한다. |
| **템플릿 초기화** | 저장된 기준 템플릿을 삭제하고 임시 기준으로 되돌린다. |

> **cf. 용어 해설**
> - **Reference (기준 영상)**: ECC 정렬 및 차영상 비교의 기준이 되는 영상. 이물이 없는 정상 상태를 촬영하여 저장한다.
> - **템플릿 (Template)**: 비교 기준으로 영구 저장된 기준 영상 및 ROI 정보. 삭제 시 시스템은 임시 기준(이전 프레임)으로 대체한다.

### 4.3 Global Motion Score

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **상한 (motion_threshold_high)** | gray level² (MSE) | 1.0~200.0 | 30.0 / 49.0 | 프레임 간 전역 MSE가 이 값 이상이면 액체 유동이 커서 검사를 건너뛴다. |
| **하한 (motion_threshold_low)** | gray level² (MSE) | 0.0~50.0 | 1.0 / 0.5 | 차영상 평균이 이 값 이하이면 거의 정지 상태로 low confidence 처리. |

> **연동**: 상한을 높이면 더 많은 움직임을 허용하여 검사가 덜 중단된다. 하한을 높이면 미세한 변화도 무시하여 정보량이 부족한 프레임에서의 오검출을 줄인다.

> **cf. 용어 해설**
> - **MSE (Mean Squared Error, 평균 제곱 오차)**: 두 영상의 대응 픽셀 밝기 차이를 제곱하여 평균한 값. 전체적인 변화량을 하나의 수치로 나타낸다.
> - **Global Motion Score**: 전체 영상 영역에서 인접 프레임 간 MSE로 측정한 움직임 지표. 액체 유동의 크기를 판별한다.
> - **차영상 (Difference Image)**: 기준 영상과 현재 영상의 픽셀별 차이를 계산한 영상. 변화가 발생한 영역이 밝게 나타난다.
> - **유동 (Flow)**: 바이알 내 액체의 흔들림이나 움직임. 유동이 크면 이물 검출의 신뢰도가 떨어지므로 검사를 건너뛴다.
> - **Low Confidence (낮은 신뢰도)**: 차영상의 변화량이 너무 적어 이물 검출에 충분한 정보가 없는 상태.

### 4.4 정렬 알고리즘

| 알고리즘 | 속도 | 회전 지원 | 특징 |
|----------|------|-----------|------|
| **ECC** | 느림 (해상도 의존) | O | 이미지 세기 기반 반복 최적화. 가장 정확하지만 큰 해상도에서 느릴 수 있다. |
| **Phase Correlate** | 매우 빠름 (30~50ms) | X (이동만) | FFT 기반 위상 상관. dx/dy만 계산하며 회전은 무시한다. |
| **ORB Feature** | 빠름 (10~30ms) | O | 특징점 매칭 + RANSAC. 회전 포함. 텍스처가 부족한 영상에서 실패 가능. |

#### ECC 정렬 세부 설정 (ECC 모드 선택 시에만 표시)

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **해상도 축소 (ecc_downscale_factor)** | 배율 | 1x / 2x / 4x / 8x | 2x / 4x | 정렬 연산 전 영상을 축소하여 속도를 높인다. 4x면 가로세로 1/4로 줄여 연산량이 약 1/16로 감소. 높을수록 빠르지만 정밀도가 미세하게 떨어진다. |
| **최대 반복 횟수 (ecc_max_iter)** | 회 | 10~5,000 | 200 / 100 | ECC 알고리즘의 반복 상한. 높이면 수렴 정밀도가 높지만 최악 시 처리 시간이 증가한다. |
| **수렴 기준 ε (ecc_epsilon)** | 무차원 (상관계수 변화) | 1e-7 ~ 0.1 | 1e-4 / 0.001 | 반복 간 상관계수 변화가 이 값 이하이면 수렴으로 판단하고 조기 종료한다. 작을수록 정밀하다. |
| **Gaussian 필터 (ecc_gauss_filt_size)** | px (홀수 커널) | 1~31 | 5 / 4 | 정렬 전 노이즈를 줄이는 가우시안 필터 크기. 정렬 안정성을 높인다. |

> **연동**: `downscale_factor`를 높이면 속도는 빨라지지만 `max_iter`와 `epsilon`의 수렴 효과가 줄어든다. 일반적으로 downscale 4x + max_iter 100 + epsilon 0.001이면 충분하다.

> **cf. 용어 해설**
> - **ECC (Enhanced Correlation Coefficient)**: 두 영상 간 밝기 상관계수를 최대화하는 반복적 정렬 알고리즘. 서브픽셀 정밀도로 기하학적 변환 행렬을 추정한다.
> - **Phase Correlate (위상 상관)**: FFT를 이용하여 두 영상의 위상 차이에서 이동량(dx, dy)을 추정하는 고속 정렬 기법. 회전은 지원하지 않는다.
> - **FFT (Fast Fourier Transform, 고속 푸리에 변환)**: 영상을 공간 영역에서 주파수 영역으로 변환하는 알고리즘. 위상 상관 및 주파수 분석에 사용된다.
> - **ORB (Oriented FAST and Rotated BRIEF)**: 빠르고 회전 불변인 특징점 검출 및 기술자 알고리즘. 실시간 응용에 적합한 경량 특징점 매칭 방법이다.
> - **RANSAC (Random Sample Consensus)**: 이상치가 포함된 데이터에서 모델을 강건하게 추정하는 반복적 알고리즘. 잘못된 특징점 매칭을 제거하는 데 사용된다.
> - **특징점 (Feature Point)**: 영상에서 코너, 블롭 등 고유한 지역 패턴을 가진 점. 영상 간 대응점 매칭에 사용된다.
> - **Warp (워프)**: 추정된 변환 행렬을 적용하여 영상의 기하학적 형태를 변형(이동, 회전, 스케일)하는 연산.
> - **Affine Transform (아핀 변환)**: 이동, 회전, 스케일, 전단(shear)을 포함하는 6 자유도의 선형 기하 변환. 평행선이 보존된다.
> - **Euclidean Transform (유클리드 변환)**: 이동과 회전만 포함하는 3 자유도의 강체 변환. 크기와 형태가 보존된다.
> - **위상 상관 (Phase Correlation)**: 두 영상의 FFT 위상 차이를 분석하여 이동량을 추정하는 기법. 밝기 변화에 강건하다.
> - **상관계수**: 두 영상의 밝기 패턴이 얼마나 유사한지를 -1~1 범위로 나타내는 수치. 1에 가까울수록 정렬이 잘 된 것이다.
> - **수렴 (Convergence)**: 반복 알고리즘에서 해가 더 이상 변하지 않는 안정 상태에 도달하는 것. 수렴 기준(ε) 이하로 변화하면 반복을 종료한다.
> - **GaussianBlur (가우시안 블러)**: 가우시안 함수를 커널로 사용하는 평활화 필터. 정렬 전 노이즈를 제거하여 알고리즘의 안정성을 높인다.
> - **ε (Epsilon, 엡실론)**: 수렴 판정을 위한 허용 오차. 반복 간 상관계수 변화가 이 값 이하이면 수렴으로 판단한다.
> - **다운스케일 (Downscale)**: 영상의 해상도를 줄이는 처리. 연산 속도를 높이지만 미세한 정밀도가 감소한다.

### 4.5 정렬 ROI

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **전체 프레임** | — | — | — | 전체 프레임을 정렬 영역으로 사용한다. 정확하지만 느리다. |
| **검사 ROI + 패딩** | — | — | — | 검사 ROI 주변만 정렬에 사용하여 속도를 높인다. |
| **좌우 패딩 (alignment_padding_x)** | px | 0~500 | 50 / 300 | 검사 ROI 좌우로 확장할 픽셀 수. 클수록 정렬 정보가 풍부해지지만 속도가 느려진다. |
| **상하 패딩 (alignment_padding_y)** | px | 0~500 | 50 / 300 | 검사 ROI 상하로 확장할 픽셀 수. |

> **연동**: 패딩 모드에서 패딩이 너무 작으면 정렬에 필요한 특징이 부족하여 실패할 수 있다. 검사 ROI 크기의 20~50%가 적절하다.

> **cf. 용어 해설**
> - **ROI (Region of Interest, 관심 영역)**: 전체 영상 중 실제 검사 또는 정렬 대상이 되는 부분 영역.
> - **패딩 (Padding)**: 검사 ROI 주변으로 추가 확장하는 여백 영역. 정렬 알고리즘에 필요한 충분한 특징 정보를 제공하기 위해 사용한다.
> - **정렬 (Alignment)**: 기준 영상과 현재 영상의 기하학적 위치를 맞추는 과정. ROI + 패딩 영역에서 수행하여 검사 영역의 정밀 정렬을 보장한다.

### 4.6 정렬·검사 영역

| 모드 | 설명 |
|------|------|
| **전체 프레임 정렬 후 ROI 검사** | 전체 프레임에 warp를 적용한 뒤 ROI 안에서만 blob 검사. 디버그 시 전체 정렬 영상 확인 가능. |
| **ROI 영역만 정렬·검사 (use_roi_only_warp)** | 검사 ROI만 warp·차영상·blob 연산. ROI가 작으면 속도가 크게 빨라진다. 결과는 동일하다. |

> **cf. 용어 해설**
> - **Warp (워프)**: 추정된 변환 행렬(이동, 회전 등)을 적용하여 영상의 픽셀 좌표를 재배치하는 기하학적 변환 연산.
> - **ROI (Region of Interest, 관심 영역)**: 전체 영상 중 실제 검사 대상이 되는 부분 영역. 불필요한 영역을 제외하여 연산량을 줄인다.

### 4.7 차영상 Sweet Spot

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **상한 (diff_too_high)** | gray level (차영상 평균) | 1.0~255.0 | 40.0 / 51.0 | 유효 마스크 영역의 차영상 평균이 이 값 이상이면 아직 유동이 크거나 정렬 실패로 판단하여 검사를 스킵한다. |
| **하한 (diff_too_low)** | gray level (차영상 평균) | 0.0~50.0 | 0.5 | 차영상 평균이 이 값 이하이면 변화가 거의 없어 low confidence로 처리한다. |

> **동작**: 정렬된 현재 프레임과 기준 프레임의 단방향 차이(`ref - current`, 즉 현재 프레임에서 새로 어두워진 부분만)를 구한 뒤, 유효 마스크 내부의 평균을 계산하여 판정한다.

> **cf. 용어 해설**
> - **Sweet Spot**: 차영상의 평균 변화량이 검사에 적합한 상한~하한 범위 내에 있는 최적 구간. 너무 크면 정렬 실패/유동, 너무 작으면 정보 부족이다.
> - **차영상 (Difference Image)**: 기준 영상과 현재 영상의 픽셀별 차이를 계산한 영상.
> - **단방향 차이 (Unidirectional Difference)**: `ref - current`처럼 한 방향의 차이만 계산하는 방식. 현재 프레임에서 새로 어두워진(이물이 나타난) 부분만 검출한다.
> - **Low Confidence (낮은 신뢰도)**: 차영상의 변화량이 너무 적어 이물 유무를 판별하기 어려운 상태. 검사 결과의 신뢰도가 낮음을 나타낸다.

### 4.8 Safe Liquid ROI

검사 ROI 내에서 유리벽, 액면, 바닥 등 노이즈가 많은 영역을 제외하고 **순수 액체 영역만** 검사 허용 구역으로 설정한다.

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **벽 제외 margin (safe_wall_margin)** | px | 0~500 | 20 | 검사 ROI 좌우 벽에서 안쪽으로 제외할 거리. 유리벽 굴절/반사 영역을 마스킹한다. |
| **상부/액면 band (safe_surface_band)** | px | 0~500 | 20 | 검사 ROI 상단 또는 추정된 액면으로부터 아래로 제외할 높이. 액면 출렁임을 마스킹한다. |
| **하부 제외 band (safe_bottom_band)** | px | 0~500 | 10 | 검사 ROI 하단에서 위로 제외할 높이. 바이알 바닥 굴절 영역 마스킹. |
| **최소 유효 면적 (safe_min_area)** | px² | 0~1,000,000 | 1,000 | 마스킹 후 남은 유효 면적이 이 값 미만이면 검사 불가(`invalid_safe_roi`)로 처리한다. |
| **액면 추정 사용 (use_surface_estimate)** | on/off | — | on | 상부 고정 band 외에 Sobel 기울기로 액면 위치를 자동 추정하여 제외 영역을 보수적으로 조정한다. |
| **액면 검색 비율 (surface_search_ratio)** | 비율 | 0.10~1.00 | 0.6 | 검사 ROI 상단에서 액면을 탐색할 높이 비율. 0.6이면 ROI 상단 60% 영역에서만 액면을 찾는다. |

> **연동**: `safe_wall_margin` + `safe_surface_band` + `safe_bottom_band` = 검사 ROI를 사방에서 축소시킨다. 너무 크게 설정하면 실제 검사 영역이 없어질 수 있으며, 이때 `safe_min_area` 검사에 걸려 검사가 스킵된다.

#### 표면 Dark 제외

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **상단 dark band 제외** | on/off | — | on | 액면 뒤쪽/상단부의 저휘도 띠를 추가로 제외한다. |
| **dark 임계값 (surface_dark_threshold)** | gray level | 0~255 | 30 | 이 밝기 이하인 픽셀을 dark band로 추출한다. |
| **dilate 커널 (surface_dark_dilate)** | px (홀수 커널) | 1~31 | 3 | 추출된 dark band를 확장(dilate)하여 제외 영역을 넓힌다. |

> **cf. 용어 해설**
> - **Safe Liquid ROI**: 검사 ROI 내에서 유리벽, 액면, 바닥 등 노이즈원을 제외한 순수 액체 검사 허용 영역.
> - **마스킹 (Masking)**: 특정 영역을 마스크로 가려서 연산이나 검출에서 제외하는 처리 기법.
> - **Sobel**: 영상의 수평/수직 방향 밝기 기울기(그래디언트)를 계산하는 미분 필터. 에지 검출 및 액면 위치 추정에 사용된다.
> - **dilate (팽창)**: 마스크 영역을 커널 크기만큼 확장하는 모폴로지 연산. 제외 영역을 넓혀 경계 부근의 노이즈까지 차단한다.
> - **연결 성분 (Connected Component)**: 이진 영상에서 서로 인접한 픽셀이 연결되어 이루는 하나의 덩어리. ROI 경계에 접촉하는 성분만 외곽 마스크로 사용한다.
> - **액면 (Liquid Surface)**: 바이알 내 액체의 상단 경계면. 출렁임이 발생하여 검출 노이즈의 원인이 된다.
> - **굴절 (Refraction)**: 빛이 유리와 액체의 경계면을 통과할 때 방향이 꺾이는 현상. 유리벽 근처에서 왜곡된 영상이 나타나는 원인이다.
> - **반사 (Reflection)**: 빛이 유리 표면에서 되돌아오는 현상. 유리벽 근처에서 밝은 반점이나 줄무늬 형태의 노이즈를 발생시킨다.

### 4.9 외곽 마스킹

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **외곽 마스킹 사용 (outer_mask_enabled)** | on/off | — | off | 정렬된 현재 영상에서 ROI 경계에 닿는 어두운 영역(유리벽)을 동적으로 제외한다. |
| **외곽 임계값 (outer_mask_threshold)** | gray level | 0~255 | 20 / 117 | 이 밝기 이하인 픽셀을 외곽 병 영역 후보로 추출. ROI 경계에 접촉하는 연결 성분만 실제 마스크가 된다. |

> **cf. 용어 해설**
> - **외곽 마스킹**: ROI 경계에 닿는 어두운 영역(유리벽 등)을 동적으로 감지하여 검사에서 제외하는 기법. 바이알 위치 변화에 자동 대응한다.
> - **연결 성분 (Connected Component)**: 이진 영상에서 서로 인접한 픽셀이 연결되어 이루는 하나의 덩어리. ROI 경계에 접촉하는 성분만 실제 외곽 마스크로 채택된다.
> - **동적 제외**: 프레임마다 실시간으로 외곽 영역을 재계산하여 바이알 위치 변동에 적응적으로 대응하는 처리 방식.

### 4.10 Blob 필터 (차영상 이물 검출)

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **이진화 임계값 (blob_threshold)** | gray level | 1~255 | 25 / 30 | 차영상에서 이 값 이상 변화한 픽셀을 blob 후보로 추출한다. |
| **최소 면적 (blob_min_area)** | px² | 1~10,000 | 10 / 1 | blob 윤곽선 면적이 이 값 미만이면 제거. |
| **최대 면적 (blob_max_area)** | px² | 1~100,000 | 5,000 / 500 | blob 면적이 이 값 초과면 제거. |
| **최소 원형도 (blob_min_circularity)** | 비율 | 0.00~1.00 | 0.0 | 0이면 형상 제한 없음. |
| **Morph Open** | px (홀수 커널) | 1~31 | 3 | 이진화 후 작은 노이즈 제거. |
| **Morph Close** | px (홀수 커널) | 1~31 | 5 / 3 | 끊어진 blob 연결. |

> **연동**: `blob_threshold`가 낮으면 미세한 차이도 후보가 되어 검출 수가 많아지지만 오검출도 증가한다. `blob_min_area`와 함께 조절하여 적절한 민감도를 찾는다.

> **cf. 용어 해설**
> - **Blob**: 이진 영상에서 연결된 흰색 픽셀의 덩어리. 차영상 이진화 후 이물 후보 영역을 나타낸다.
> - **이진화 (Binarization)**: 차영상의 각 픽셀을 임계값 기준으로 0 또는 255 두 값으로 변환하는 과정. 이물 후보 영역을 분리한다.
> - **차영상 (Difference Image)**: 기준 영상과 현재 영상의 픽셀별 차이를 계산한 영상.
> - **Morphology (형태학 연산)**: 구조 요소를 이용하여 이진 영상의 형태를 변형하는 연산. 노이즈 제거(Opening)와 영역 연결(Closing)에 사용된다.
> - **Opening (열림)**: 침식 후 팽창. 작은 점 노이즈를 제거한다.
> - **Closing (닫힘)**: 팽창 후 침식. 끊어진 blob 조각을 연결한다.
> - **Contour (윤곽선)**: 이진 영상에서 blob의 외곽 경계를 이루는 점들의 집합. 면적, 원형도 등 형상 지표 계산에 사용된다.
> - **Circularity (원형도)**: `4π × Area / Perimeter²`로 계산되는 형상 지표. 0이면 형상 제한 없이 모든 형태를 허용한다.

### 4.11 다프레임 통합 (Temporal Tracking)

단일 프레임의 일시적 노이즈를 배제하기 위해, **여러 프레임에서 반복 관찰된** 이물만 최종 불량으로 판정하는 추적 시스템이다.

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **연속 프레임 후보 통합 (track_enabled)** | on/off | — | on | 활성화 시 단일 프레임 blob를 바로 defect로 처리하지 않고, 반복 관찰된 후보만 유지한다. |
| **기록 프레임 수 (track_history_frames)** | 프레임 | 1~100 | 6 | 후보 track을 유지할 최대 프레임 수. 이 프레임 수 동안 재관찰되지 않으면 track이 소멸한다. |
| **연결 허용 거리 (track_match_distance)** | px (유클리드 거리) | 0.0~500.0 | 18.0 | 서로 다른 프레임의 후보를 같은 track으로 연결할 최대 중심점 거리. 이물이 미세하게 움직여도 동일 이물로 인식하는 범위. |
| **최소 재검출 횟수 (track_min_hits)** | 회 | 1~100 | 2 | 이 횟수 이상 연속 검출되어야 최종 defect로 확정. 1이면 트래킹 효과 없음(단일 프레임 즉시 확정). |
| **면적 변화 허용비 (track_max_area_ratio)** | 무차원 (배수) | 1.0~20.0 | 3.0 | 같은 track으로 연결할 때 `max(면적1, 면적2) / min(면적1, 면적2)`가 이 값 이내여야 한다. 이물 면적이 프레임마다 급변하면 별개 track으로 분리된다. |

> **연동**: `track_min_hits`를 높이면 오검출이 줄어들지만, 짧은 정지 구간에서 실제 이물을 놓칠 수 있다. `track_history_frames`와 `track_min_hits`의 관계: history 내에 min_hits 이상 관찰되어야 하므로, `min_hits ≤ history_frames`이어야 의미가 있다.

> **cf. 용어 해설**
> - **Temporal Tracking (시간적 추적)**: 여러 프레임에 걸쳐 동일한 후보를 시간 축으로 추적하여 일시적 노이즈와 실제 이물을 구별하는 기법.
> - **유클리드 거리 (Euclidean Distance)**: 두 점 사이의 직선 거리. `√((x1-x2)² + (y1-y2)²)`로 계산되며, 다른 프레임의 후보가 같은 이물인지 판별하는 기준이다.
> - **Track (트랙)**: 여러 프레임에 걸쳐 동일한 이물로 연결된 후보의 이력(궤적). 각 track은 검출 횟수(hit)와 위치 정보를 유지한다.
> - **Hit (히트)**: 특정 track이 한 프레임에서 재검출된 횟수. 누적 hit 수가 min_hits 이상이면 최종 defect로 확정된다.
> - **다프레임 (Multi-frame)**: 여러 연속 프레임에 걸친 정보를 종합하여 판단하는 방식. 단일 프레임의 일시적 노이즈를 걸러낸다.
> - **Defect (결함)**: 최종적으로 불량으로 확정된 검출 객체. 트래킹을 통해 반복 관찰이 확인된 이물을 의미한다.

## 5. 탭 4: DL 최적화 (Deep Learning Optimization)

AI 모델 추론 속도를 극대화하기 위한 하드웨어 가속 설정이다.

| 파라미터 | 범위 | 기본값 | 동작 |
|----------|------|--------|------|
| **Optimization Level** | 0~5 | 4 | Level 0~1: 기본 CPU/GPU. Level 2~4: `torch.compile`, GPU Crop/Resize, Grid Sample 등 파이프라인 병목 제거. Level 5: OpenVINO 추론 엔진 사용. |
| **OpenVINO Device** | CPU / GPU / NPU | CPU | Level 5에서 사용할 인텔 하드웨어. NPU(Neural Processing Unit)는 최신 인텔 프로세서에서 저전력/고효율 추론을 수행한다. |

> **cf. 용어 해설**
> - **Optimization Level (최적화 레벨)**: AI 모델 추론의 하드웨어 가속 수준을 단계적으로 설정하는 파라미터. 높을수록 더 공격적인 최적화를 적용한다.
> - **OpenVINO**: 인텔에서 개발한 딥러닝 추론 최적화 툴킷. 인텔 CPU, GPU, NPU에서 모델 추론 성능을 극대화한다.
> - **NPU (Neural Processing Unit, 신경망 처리 장치)**: 딥러닝 연산에 특화된 전용 프로세서. 최신 인텔 프로세서에 내장되어 저전력으로 고효율 AI 추론을 수행한다.
> - **torch.compile**: PyTorch의 JIT 컴파일러. 모델 그래프를 최적화하여 추론 속도를 향상시키는 파이썬 레벨의 가속 기법.
> - **Grid Sample**: GPU에서 영상의 기하학적 변환(워프, 리사이즈 등)을 수행하는 텐서 연산. CPU 대비 대폭적인 속도 향상을 제공한다.

### 6.1 공통 및 기존 검사
| 뷰 | 의미 | 관련 파라미터 |
|----|------|--------------|
| **최종 결과 (Result)** | 원본 위에 검출 윤곽선과 라벨을 오버레이 | 모든 파라미터의 최종 결과 |
| **원본 (Gray)** | 전처리 전 그레이스케일 입력 영상 | — |

| **Opening** | 작은 노이즈가 제거된 후 영상 | `Open 커널` |
| **Closing** | 끊어진 이물이 연결된 후 영상 | `Close 커널` |

> **cf. 용어 해설**
> - **오버레이 (Overlay)**: 원본 영상 위에 검출 결과(윤곽선, 라벨, 바운딩박스 등)를 반투명하게 겹쳐 표시하는 시각화 방식.
> - **그레이스케일 (Grayscale)**: 컬러 정보를 제거하고 밝기 정보만 남긴 단일 채널 영상. 0(검정)~255(흰색) 범위로 표현된다.
> - **이진화 (Binarization)**: 그레이스케일 영상의 각 픽셀을 임계값 기준으로 0 또는 255 두 값으로 변환하는 과정.

### 6.2 Bubble 검출용
| 뷰 | 의미 | 관련 파라미터 |
|----|------|--------------|
| **평탄화+CLAHE (Flat)** | 배경 편차 제거 + CLAHE 보정 결과 | `bg_open_ksize`, `bg_smooth_sigma`, CLAHE 설정 |
| **DoG 밴드패스 (Diff)** | 두 Gaussian 차이로 기포 신호 강조 | `sigma_small`, `sigma_large` |
| **MAD 이진화 (Binary)** | DoG에 적응형 임계값 적용 결과 | `thr_k` |
| **Bubble 최종** | 형상 필터 통과한 최종 기포만 표시 | `min/max_diameter`, `circularity_min`, `solidity_min` |

> **cf. 용어 해설**
> - **평탄화 (Flattening)**: 배경 밝기 편차를 제거하여 균일한 밝기 분포를 만드는 전처리.
> - **CLAHE**: 지역적 히스토그램 평활화에 대비 제한을 적용한 대비 향상 기법.
> - **DoG (Difference of Gaussians)**: 두 가우시안 블러 결과의 차이로 특정 크기 범위의 구조를 강조하는 밴드패스 필터.
> - **MAD 이진화**: MAD 기반 적응 임계값으로 DoG 결과를 이진화하는 과정. 배경 통계에 자동 적응한다.
> - **형상 필터**: 원형도, 충실도, 종횡비, 직경 등 기하학적 조건으로 후보를 걸러내는 필터.

### 6.3 ECC 정렬용
| 뷰 | 의미 | 관련 파라미터 |
|----|------|--------------|
| **ECC 정렬 프레임** | 기준에 맞춰 warp된 현재 프레임 | 정렬 알고리즘 설정 |
| **ECC 차영상 (AbsDiff)** | 정렬된 영상과 기준의 절대 차이 | — |
| **ECC 이진화** | 차영상에서 blob_threshold 이상 변화 부분 | `blob_threshold` |
| **ECC Motion 입력** | 인접 프레임 간 순수 움직임 차이 맵 | — |
| **ECC 정렬 ROI** | 정렬 시 참조하는 영역 (녹색 사각형) | `alignment_mode`, `padding` |
| **ECC 마스킹** | 검사 제외 영역 (외곽 병 등) | `outer_mask_*` |
| **ECC Safe ROI** | 모든 마스킹 반영 후 최종 유효 검사 영역 | `safe_*` 전체 |

> **cf. 용어 해설**
> - **Warp (워프)**: 추정된 변환 행렬을 적용하여 영상의 기하학적 형태를 변형하는 연산.
> - **AbsDiff (절대 차이)**: 정렬된 두 영상의 픽셀별 밝기 차이의 절대값. 변화가 발생한 영역이 밝게 나타난다.
> - **이진화 (Binarization)**: 차영상을 임계값 기준으로 0/255로 변환하여 이물 후보 영역을 분리하는 과정.
> - **Motion 입력**: 인접 프레임 간의 순수 움직임 차이를 시각화한 맵. 상태 게이트의 Motion Score 산출에 사용된다.
> - **정렬 ROI**: 정렬 알고리즘이 참조하는 영역. 패딩을 포함하여 충분한 특징 정보를 제공한다.
> - **마스킹 (Masking)**: 외곽 병 영역 등 검사에서 제외할 영역을 가리는 처리.
> - **Safe ROI**: 모든 마스킹(벽, 액면, 바닥, 외곽)이 반영된 후 남은 최종 유효 검사 영역.

## 7. 로그 및 상태 메시지 가이드

| 메시지 | 의미 |
|--------|------|
| **"검사 중..." (Amber)** | 파라미터 변경 후 백그라운드에서 연산 수행 중. |
| **"Noise: 2, Particle: 1..." (Green)** | 검출 완료. 각 분류별 개수와 소요 시간(ms) 출력. |
| **"ECC size_mismatch" (Orange)** | 기준 이미지와 현재 프레임의 해상도가 달라 정렬 불가. |
| **"ECC no_reference" (Gray)** | 기준 이미지가 없어 수정 검사를 수행할 수 없는 상태. |
| **"Position: (X, Y) Gray: V"** | 뷰어의 마우스 위치 정보 실시간 추적. |

> **cf. 용어 해설**
> - **Size Mismatch (크기 불일치)**: 기준 영상(Reference)과 현재 프레임의 해상도(가로×세로 픽셀 수)가 달라 정렬 연산을 수행할 수 없는 오류 상태.
> - **No Reference (기준 없음)**: ECC 정렬 검사를 위한 기준 영상이 저장되어 있지 않은 상태. 기준 템플릿을 저장해야 검사가 가능하다.
> - **Noise**: 검출된 이물 중 대비가 낮아 Noise_Dust로 분류된 객체의 수.
> - **Particle**: 검출된 이물 중 대비가 높아 Particle로 분류된 객체의 수.
> - **ms (밀리초)**: 1/1000초 단위의 시간. 검사 소요 시간을 표시하는 데 사용된다.
