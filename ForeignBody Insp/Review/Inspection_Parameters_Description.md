# 검사 파라미터 상세 설명서 (Inspection Parameters Description)

본 문서는 바이알 이물 검사 시스템의 핵심 설정을 담당하는 **Inspection Parameters** 창의 모든 기능, 파라미터 및 디바운싱 로직에 대해 상세히 기술합니다. 본 설정창은 실시간 디버깅 뷰와 연동되어 파라미터 변경에 따른 검출 결과를 즉각적으로 확인할 수 있도록 설계되었습니다.

---

## 1. 공통 제어 및 디버깅 뷰 (Common Controls & View)

설정창 상단과 왼쪽 영역은 실시간 피드백을 위한 제어 도구들로 구성됩니다.

### 1.1 검사 모드 선택 (Inspection Mode)
*   **기존 검사 (Legacy)**: 단일 프레임 기반의 Threshold 및 형태학적 특징 추출 방식입니다. 정지된 상태의 이미지 분석에 최적화되어 있습니다.
*   **수정 검사 (ECC)**: 연속 프레임 간의 정렬(Alignment) 및 정적 배경과의 차영상(AbsDiff)을 이용한 방식입니다. 액체 유동이 있는 동적 환경에서 미세 이물을 검출하는 데 강력합니다.

### 1.2 실시간 디버깅 제어
*   **실시간 디버깅**: 체크 시 파라미터 변경 즉시 백그라운드 스레드에서 유효성 검사를 수행하고 결과를 뷰에 업데이트합니다.
*   **파라미터별 View 표시**: 체크 시 특정 파라미터를 조절하면 해당 연산 단계의 중간 이미지(예: Threshold 조절 시 이진화 영상)로 자동 전환됩니다.
*   **표시 (Debug View Selection)**: 원본 Gray부터 최종 결과(Result), 중간 단계(Opening, Closing, Diff Map 등) 총 16가지 뷰를 선택할 수 있습니다.
*   **맞춤 버튼**: 이미지를 현재 뷰 크기에 맞게 배율을 조정합니다.
*   **상태바 (Footer)**: 마우스 커서 위치의 픽셀 좌표(X, Y)와 해당 위치의 밝기값(GrayValue) 또는 RGB 값을 실시간으로 표시합니다.

## 2. 탭별 파라미터 상세 (Tab Details)

### 2.1 분류/검출 조건 (Classification/Detection)
일반적인 이물 검출 및 룰 기반 분류를 설정합니다.

| 그룹 | 파라미터 | 상세 설명 및 영향 |
| :--- | :--- | :--- |
| **상태 게이트** | 정지 상태 확정 후 검사 | 8대 상태 머신(running, settling, stop_confirmed 등)이 확정된 프레임에서만 검사를 수행할지 여부입니다. |
| **정지 판단** | Stop Motion/Edge/Dark | 정지 상태로 진입하기 위한 상한선입니다. 이 값들보다 변화량이 작아야 '정지 후보'가 됩니다. |
| **정지 판단** | Reenter Motion/Edge/Dark | 다시 '운행(Running)' 상태로 복귀시키기 위한 하한선입니다. 진동이나 유동이 발생하면 이 값을 기준으로 감지합니다. |
| **정지 판단** | Candidate/Confirm Frames | 상태 전환을 확정하기 위해 필요한 연속 안정 프레임 수입니다. |
| **디버그 필터** | VIEW ALL / NOISE / PARTICLE | 뷰에 표시할 이물 종류를 선택합니다. |
| **분류 조건** | Noise/Particle 대비 임계값 | 이물과 배경의 밝기 차이(Contrast)가 이 값 이상이면 Particle, 미만이면 Noise_Dust로 분류합니다. |
| **Threshold** | Threshold (보조) | 이진화 밝기 기준값입니다. Adaptive Threshold 사용 시 로컬 임계값의 가이드 역할을 합니다. |
| **영역 필터** | Min / Max Area | 검출할 이물의 최소/최대 면적(px²) 제한입니다. 너무 작은 노이즈나 너무 큰 구조물을 제외합니다. |
| **형상 필터** | Circularity / Solidity / Aspect | 원형도, 볼록도, 종횡비 제한입니다. 실이나 기스크 등 특정 형태의 오검출을 방지합니다. |
| **모폴로지** | Open / Close 커널 | 노이즈 제거(Open) 및 끊어진 이물 연결(Close)을 위한 커널 크기입니다. |

### 2.2 Bubble 검출 (Bubble Detection)
DoG(Difference of Gaussians) 알고리즘을 이용해 액체 내 기포를 특화 검출합니다.

*   **배경 평탄화 (Morph Opening)**: 커널 크기를 크게 설정(예: 101)하여 바이알의 곡면 배경만 추출한 후 원본에서 감산해 배경 편차를 제거합니다.
*   **노이즈 제거**: Median 또는 Bilateral 필터를 통해 DoG 연산 전 미세 잡음을 억제합니다.
*   **DoG 밴드패스**: σ small(최소 두께)과 σ large(최대 두께)의 차이를 통해 특정 크기 범위의 기포 신호만 강조합니다.
*   **MAD 적응 임계**: 중앙값 절대 편차(MAD) 기반으로 주변 변동폭의 `k`배 이상인 픽셀만 기포 후보로 선정하여 조명 변화에 강인하게 대응합니다.
*   **형상 필터**: 최소/최대 직경(Diameter)과 원형도(0.35 이상 권장)를 통해 기포 특유의 둥근 형태를 확정합니다.

### 2.3 ECC 정렬 검사 (ECC Alignment)
고급 정렬 알고리즘을 사용한 정밀 검사 설정입니다.

*   **정렬 알고리즘**:
    *   **ECC**: 이미지 세기 기반의 반복 최적화 (가장 정확, 느림).
    *   **Phase Correlate**: FFT 기반 위상 상관 (초고속, 이동량만 계산).
    *   **ORB Feature**: 특징점 매칭 (회전 포함, 빠름).
*   **정렬 ROI**: 전체 프레임을 다 쓸지, 검사 ROI 주변만 쓸지 선택하여 연산 속도를 조절합니다.
*   **다프레임 통합 (Tracking)**:
    *   **연속 프레임 후보 통합**: 일시적인 노이즈를 배제하기 위해 여러 프레임에서 반복 관찰된(Min Hits) 이물만 최종 불량으로 판정합니다.
    *   **연결 허용 거리**: 프레임 간 이물이 움직였을 때 동일 이물로 간주할 거리 범위입니다.
*   **Safe Liquid ROI**:
    *   **벽/액면 제외 Margin**: 유리벽의 굴절이나 액면의 출렁임 영역을 검사 제외 구역(Mask)으로 자동 설정합니다.

### 2.4 DL 최적화 (DL Optimization)
AI 모델 추론 속도를 극대화하기 위한 하드웨어 가속 설정입니다.

*   **Optimization Levels**:
    *   **Level 0~1**: 기본 CPU/GPU 파이프라인.
    *   **Level 2~4**: `torch.compile`, GPU 전용 Crop/Resize, 격자 샘플링(Grid Sample) 등을 통한 파이프라인 병목 제거.
    *   **Level 5 (OpenVINO)**: 인텔 하드웨어(CPU, iGPU, NPU)에 최적화된 추론 엔진 사용.
*   **OpenVINO 장치**: 최신 인텔 프로세서의 **NPU(Neural Processing Unit)**를 선택하여 저전력/고효율 검사를 수행할 수 있습니다.

## 3. 디버그 뷰(표시) 항목 상세 설명 (Debug View Items)

상단 **표시** 콤보박스에서 선택 가능한 각 단계별 영상의 기술적 의미와 목적입니다.

### 3.1 공통 및 기존 검사 (Common & Legacy)
*   **최종 결과 (Result)**: 모든 검사(일반, 버블, ECC)가 완료된 후, 원본 영상 위에 검출된 이물의 윤곽선과 라벨(Particle, Bubble 등)을 오버레이하여 보여줍니다.
*   **원본 (Gray)**: 전처리가 가해지지 않은 순수 그레이스케일 입력 영상입니다.
*   **Threshold**: 일반 검출 설정의 임계값(Threshold)이 적용된 이진화 영상입니다. 이물이 흰색(또는 검은색)으로 분리되는지 확인합니다.
*   **Opening**: 작은 점 형태의 노이즈가 제거된 후의 영상입니다. 'Open 커널' 파라미터의 영향을 받습니다.
*   **Closing**: 끊어진 이물 조각들이 하나로 합쳐지고 내부 구멍이 메워진 결과입니다. 'Close 커널' 파라미터의 영향을 받습니다.

### 3.2 Bubble 검출용 (For Bubble Detection)
*   **평탄화+CLAHE (Flat)**: 바이알 배경의 밝기 편차를 제거하고(Morph Opening), CLAHE 보정까지 마친 전처리 영상입니다.
*   **DoG 밴드패스 (Diff)**: 두 가우시안 블러의 차이(Difference of Gaussians)를 구한 결과입니다. 기포 후보가 밝은 신호로 강조됩니다.
*   **MAD 이진화 (Binary)**: DoG 결과에 적응형 임계값(MAD)을 적용하여 기포 후보를 0과 1로 분리한 영상입니다.
*   **Bubble 최종**: 형태학적 정리와 형상 필터(직경, 원형도 등)를 거쳐 최종적으로 '기포'로 확정된 객체들만 보여줍니다.

### 3.3 ECC 정렬 및 수정 검사용 (For ECC Alignment)
*   **ECC 정렬 프레임**: 기준(Ref) 이미지와 일치하도록 수학적으로 비틀기(Warp) 연산이 적용된 현재 프레임입니다. 정렬이 잘 되었는지 육안으로 확인하는 핵심 뷰입니다.
*   **ECC 차영상 (AbsDiff)**: 정렬된 현재 영상과 기준 영상의 절댓값 차이(Absolute Difference)입니다. 이물이나 액체 유동이 있는 부분만 밝게 나타납니다.
*   **ECC 이진화**: 차영상에서 '이진화 임계값' 이상으로 변한 부분을 추출한 결과입니다.
*   **ECC Motion 입력**: 프레임 간의 순수 움직임(Global Motion)을 계산하기 위해 사용되는 차이 맵입니다. 안정 상태 판별용 지표로 쓰입니다.
*   **ECC 정렬 ROI**: 정렬 연산 시 실제로 참고하는 영역을 녹색 사각형으로 표시합니다. (ROI + 패딩 설정 확인용)
*   **ECC 마스킹**: 병의 외곽선이나 굴절 영역 등 검사에서 제외될 영역(Mask)을 확인합니다.
*   **ECC Safe ROI**: 마스킹과 액면 추정 등을 모두 반영하여, **실제로 이물 검사가 허용된 최종 유효 영역**을 시각화합니다.

## 4. 로그 및 상태 메시지 가이드 (Logs & Status)

설정창 우측 하단 및 메인 화면 로그창에 표시되는 주요 메시지의 의미입니다.

1.  **"검사 중..." (Amber)**: 파라미터 변경 후 백그라운드 스레드에서 연산이 수행 중인 상태입니다.
2.  **"Noise: 2, Particle: 1..." (Green)**: 검출 완료 후 각 분류별 개수와 소요 시간(ms)을 출력합니다.
3.  **"ECC size_mismatch" (Orange)**: 기준 이미지(Reference)와 현재 프레임의 해상도가 달라 정렬을 수행할 수 없을 때 표시됩니다.
4.  **"ECC no_reference" (Gray)**: 기준 이미지가 캡처되지 않아 수정 검사 모드를 동작시킬 수 없는 상태입니다.
5.  **"Position: (X, Y) Gray: V"**: 뷰어의 현재 마우스 위치 정보를 실시간으로 추적합니다.

---

> [!TIP]
> **디바운스(Debounce)** 기능이 적용되어 있어, 스핀박스를 빠르게 클릭하더라도 마지막 입력 후 약 200ms 대기 후에 검사가 시작되므로 UI가 멈추지 않고 쾌적하게 동작합니다.
