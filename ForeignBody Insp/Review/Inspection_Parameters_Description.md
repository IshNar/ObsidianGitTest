# 검사 파라미터 상세 설명서 (Inspection Parameters Description)

본 문서는 바이알 이물 검사 시스템의 **Inspection Parameters** 설정창에 포함된 모든 파라미터의 **단위, 범위, 기본값, 동작 원리, 상호 연동 관계**를 정확하게 기술합니다.

> **용어 규칙**
> - **px** = 이미지 픽셀 (해상도에 따라 실제 크기가 달라짐)
> - **px²** = 픽셀 면적 (윤곽선 내부 픽셀 수)
> - **gray level** = 0~255 범위의 8비트 밝기 값
> - **비율** = 0.0~1.0 범위의 무차원 계수
> - **프레임** = 카메라/동영상의 한 장 단위

---

## 1. 공통 제어 및 디버깅 뷰 (Common Controls & View)

### 1.1 검사 모드 선택 (Inspection Mode)
| 모드                 | 설명                                                                      |
| ------------------ | ----------------------------------------------------------------------- |
| **기존 검사 (Legacy)** | 단일 프레임 Threshold + 형태학 기반. 정지된 이미지에서 밝기 차이로 이물을 검출한다.                   |
| **수정 검사 (ECC)**    | 연속 프레임 정렬(Alignment) + 차영상(AbsDiff) 기반. 기준 프레임 대비 변화만 추출하여 미세 이물을 검출한다. |

두 모드는 **정지 인식(State Gate)** 로직을 공유하며, 정지가 확인된 프레임에서만 검사를 수행한다.

### 1.2 실시간 디버깅 제어
| 항목 | 설명 |
|------|------|
| **실시간 디버깅** | 체크 시 파라미터 변경 즉시 백그라운드에서 검사를 수행하고 결과를 뷰에 업데이트한다. |
| **파라미터별 View 표시** | 체크 시 특정 파라미터 조절 시 해당 연산 단계의 중간 이미지로 자동 전환된다. |
| **표시 (Debug View)** | 원본 Gray ~ 최종 결과(Result), 중간 단계(Opening, Closing, Diff Map 등) 총 16가지 뷰 선택 가능. |
| **맞춤 버튼** | 이미지를 현재 뷰 크기에 맞게 배율 조정한다. |
| **상태바 (Footer)** | 마우스 커서 위치의 픽셀 좌표(X, Y)와 밝기값(0~255)을 실시간 표시한다. |

> **디바운스(Debounce)**: 스핀박스를 빠르게 클릭해도 마지막 입력 후 약 200ms 대기 후에 검사가 시작되므로 UI가 멈추지 않는다.

## 2. 탭 1: 분류/검출 조건 (Classification/Detection)

이 탭은 **기존 검사(Legacy) 모드**의 일반 이물 검출 파라미터와, 양 모드 공유인 **상태 게이트**를 설정한다.

### 2.1 상태 게이트 (State Gate) — 정지 인식

바이알이 정지했는지 판단하는 상태 머신이다. **기존 검사 / ECC 검사 모두** 이 로직을 공유한다.

#### 동작 원리

매 프레임마다 3가지 변화량을 계산한다:

| 변화량 | 계산 방법 | 단위 | 의미 |
|--------|-----------|------|------|
| **Motion Score** | 인접 프레임 간 ROI 영역의 MSE (Mean Squared Error) | gray level² | 전체 밝기 변화 크기. 바이알 이동 시 급격히 상승한다. |
| **Edge Change** | 인접 프레임 간 Canny 에지맵의 absdiff 평균 | gray level (0~255 스케일) | 에지(윤곽) 변화량. 바이알 테두리 진입/이탈에 민감하다. |
| **Dark Change** | 인접 프레임 간 어두운 영역(outer_mask_threshold 이하) 면적 비율의 절대 차 | 비율 (0.0~1.0) | 바이알 본체가 ROI에 들어오거나 나갈 때 크게 변한다. |

#### 상태 전이 규칙

```
running → settling → stop_candidate → stop_confirmed
```

| 조건 | 판정 | 결과 |
|------|------|------|
| Motion ≥ Reenter Motion **OR** Edge ≥ Reenter Edge **OR** Dark ≥ Reenter Dark | **운행(Running)** | 안정 카운트 리셋, 검사 스킵 |
| Motion ≤ Stop Motion **AND** Edge ≤ Stop Edge **AND** Dark ≤ Stop Dark | **안정(Stable)** | 안정 카운트 +1 |
| 안정 카운트 ≥ Confirm Frames | **정지 확정(stop_confirmed)** | 실제 검사 수행 |
| Candidate Frames ≤ 안정 카운트 < Confirm Frames | **정지 후보(stop_candidate)** | 추가 확인 중 |
| 위 어느 것도 아님 | **안정화 중(settling)** | 안정 카운트 리셋 |

#### 파라미터 상세

| 파라미터 | 단위 | 범위 | 기본값 (Legacy/ECC) | 의미 |
|----------|------|------|---------------------|------|
| **정지 motion 상한** | gray level² | 0.0~255.0 | 3.0 / 5.0 | 이 값 **이하**여야 "안정"으로 누적. 낮을수록 엄격한 정지 판정. |
| **정지 edge 상한** | gray level | 0.0~255.0 | 4.0 / 6.0 | Canny 에지 변화가 이 값 이하여야 안정. |
| **정지 dark 변화** | 비율 | 0.000~1.000 | 0.02 / 0.02 | 어두운 영역 비율 변화가 이 값 이하여야 안정. 0.02 = 2% 이내. |
| **재진입 motion 상한** | gray level² | 0.0~255.0 | 12.0 / 12.0 | 이 값 **이상**이면 즉시 running으로 복귀. Stop 상한보다 높아야 한다. |
| **재진입 edge 상한** | gray level | 0.0~255.0 | 12.0 / 12.0 | 에지 변화가 다시 커지면 running 복귀. |
| **재진입 dark 변화** | 비율 | 0.000~1.000 | 0.05 / 0.05 | dark 비율 변화가 다시 커지면 running 복귀. |
| **후보 연속 프레임** | 프레임 | 1~100 | 2 / 2 | 안정 상태가 이 횟수만큼 연속되면 stop_candidate. |
| **확정 연속 프레임** | 프레임 | 1~100 | 4 / 4 | 안정 상태가 이 횟수만큼 유지되면 stop_confirmed(검사 시작). |

> **연동 관계**: `Stop 상한 < Reenter 상한` 이어야 히스테리시스가 동작한다. 두 값이 같으면 채터링(상태 진동)이 발생할 수 있다.
> `Candidate Frames < Confirm Frames`이어야 하며, Confirm Frames가 클수록 정지 확정까지 시간이 오래 걸리지만 오판이 줄어든다.

### 2.2 일반 이물 검출 파라미터

기존 검사(Legacy) 모드에서 Gray → Blur → Threshold → Morphology → Contour 순서로 사용된다.

| 파라미터 | 단위 | 범위 | 기본값 | 동작 및 연동 |
|----------|------|------|--------|-------------|
| **Threshold** | gray level | 0~255 | 100 (코드) / 220 (현재 설정) | 이진화 기준 밝기. 이 값보다 **어두운** 픽셀이 이물 후보가 된다. `Adaptive Threshold` 사용 시 보조 역할(global dark mask)을 한다. |
| **Adaptive Threshold** | on/off | — | on | 체크 시 `cv2.adaptiveThreshold(blockSize=15, C=3)` + 위 Threshold의 global dark mask를 AND 결합한다. 조명이 불균일한 바이알 곡면에 효과적. |
| **Min Area** | px² | 0~5,000 | 10 | 윤곽선 면적이 이 값 미만이면 노이즈로 제거. 값을 높이면 작은 이물을 놓칠 수 있다. |
| **Max Area** | px² | 1~100,000 | 5,000 (코드) / 500 (현재 설정) | 윤곽선 면적이 이 값 초과면 구조물로 제거. 바이알 전체 영역이 잡히는 것을 방지. |
| **Min Circularity** | 비율 | 0.00~1.00 | 0.0 | `4π × Area / Perimeter²`. 0이면 제한 없음. 1.0이면 완전 원만 통과. 길쭉한 스크래치를 걸러낼 때 0.3~0.5 사용. |
| **Min Solidity** | 비율 | 0.00~1.00 | 0.0 | `Area / ConvexHullArea`. 윤곽선 내부에 빈 공간이 많으면 낮아진다. 불규칙 노이즈 필터링용. |
| **Max Aspect Ratio** | 무차원 | 1.0~50.0 | 10.0 | `max(w,h) / min(w,h)`. 바운딩박스 장축/단축 비. 값이 클수록 길쭉한 객체도 허용한다. |
| **Open 커널** | px (커널 한 변 크기) | 1~15 | 2 | 모폴로지 Opening(침식→팽창) 커널. 이진화 후 작은 점 노이즈를 제거한다. 값이 클수록 더 큰 노이즈까지 제거하지만, 작은 실제 이물도 사라질 수 있다. |
| **Close 커널** | px (커널 한 변 크기) | 1~15 | 3 | 모폴로지 Closing(팽창→침식) 커널. 끊어진 이물 조각을 연결하고 내부 구멍을 메운다. 값이 클수록 이물이 합쳐지지만 가까운 별개 이물도 붙을 수 있다. |

> **처리 흐름**: `Gray → GaussianBlur(3,3) → [Adaptive/Static] Threshold → Opening(open_kernel) → Closing(close_kernel) → findContours → 면적/형상 필터`
>
> **연동 관계**: `Threshold`↔`Adaptive Threshold` — Adaptive가 켜지면 Threshold는 global dark mask 역할만 한다. `Min/Max Area`는 이후 형상 필터(Circularity, Solidity, Aspect)와 **AND 조건**으로 모두 통과해야 검출된다.

### 2.3 분류 조건

| 파라미터 | 단위 | 범위 | 기본값 | 의미 |
|----------|------|------|--------|------|
| **Noise/Particle 대비 임계값** | gray level | 0~255 | 10 (현재 설정) / 30 (코드 기본) | 검출된 이물 영역의 평균 밝기 대비(Contrast)가 이 값 이상이면 **Particle**, 미만이면 **Noise_Dust**로 분류한다. 룰 기반 분류에서만 사용된다. |
| **디버그 필터** | 선택 | ALL / NOISE / PARTICLE | ALL | 디버그 뷰에 표시할 이물 종류를 선택한다. 검출 자체에는 영향 없다. |

## 3. 탭 2: Bubble 검출 (Bubble Detection)

DoG(Difference of Gaussians) 알고리즘으로 액체 내 기포를 특화 검출한다. 기존 검사(Legacy)에서만 사용되며, 일반 검출과 **병렬로 동시 실행** 후 결과를 병합한다.

### 3.1 처리 파이프라인

```
배경 평탄화(Morph Opening) → 양극성 차이 → [CLAHE] → 노이즈 제거 → DoG 밴드패스 → MAD 적응 임계 → 형태학 정리 → 형상 필터
```

### 3.2 배경 평탄화

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **배경 커널 (bg_open_ksize)** | px (홀수 강제) | 3~999 | 61 / 20 | Morphological Opening으로 바이알 곡면의 큰 배경 구조만 추출하는 커널 크기. 바이알 직경/곡률에 맞춰 설정한다. 큰 값일수록 더 넓은 배경을 추정한다. |
| **배경 스무딩 σ (bg_smooth_sigma)** | px (σ값) | 0.0~ | 15.0 / 2.0 | 배경 추정 후 Gaussian 블러의 표준편차. 0이면 자동으로 `ksize/4` 적용. 값이 클수록 배경이 더 매끄러워진다. |

> **동작**: 원본에서 배경을 빼면(`flat = max(bg-work, work-bg)`) 배경 밝기 편차가 제거되어 기포 신호만 남는다.

### 3.3 CLAHE (선택적 대비 보정)

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **CLAHE 사용** | on/off | — | off | 평탄화된 영상에 CLAHE 대비 보정을 추가 적용한다. |
| **Clip Limit** | 무차원 | 0.1~ | 2.0 / 20.0 | 히스토그램 클리핑 한계. 높을수록 대비 증가가 강하다. |
| **Grid Size** | px (타일 한 변) | 1~ | 8 / 64 | CLAHE 타일 격자 크기. 작을수록 지역적 보정이 세밀하지만 노이즈 증폭 위험. |

### 3.4 노이즈 제거

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **Denoise Mode** | 선택 | median / bilateral / none | median / none | DoG 연산 전 미세 잡음을 억제하는 필터 종류. |
| **Median 커널** | px (홀수) | 3~ | 5 / 3 | Median 블러 커널 크기. 작은 솔트&페퍼 노이즈 제거에 효과적. |
| **Bilateral d** | px | 1~ | 7 | Bilateral 필터의 이웃 직경. |
| **Bilateral σColor** | gray level | 0.0~ | 25.0 | 색상 공간에서의 필터 σ. 값이 크면 밝기 차가 큰 픽셀도 혼합된다. |
| **Bilateral σSpace** | px | 0.0~ | 25.0 | 좌표 공간에서의 필터 σ. 값이 크면 먼 픽셀까지 영향을 미친다. |

### 3.5 DoG 밴드패스

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **σ small** | px (σ값) | 0.1~ | 1.2 / 5.0 | 작은 Gaussian 블러의 σ. 검출할 기포의 **최소 두께**에 대응. |
| **σ large** | px (σ값) | σ small 이상 | 6.0 / 20.0 | 큰 Gaussian 블러의 σ. 검출할 기포의 **최대 두께**에 대응. |

> **동작**: `DoG = |GaussianBlur(σ_small) - GaussianBlur(σ_large)|`
> σ_small~σ_large 사이 크기의 구조만 강조된다. 기포보다 작은 노이즈와 기포보다 큰 배경 구조가 동시에 억제된다.
> **연동**: σ_small이 작을수록 미세 기포까지 잡지만 노이즈도 증가. σ_large가 클수록 큰 기포까지 커버하지만 배경 간섭이 늘어난다.

### 3.6 MAD 적응 임계

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **thr_k (MAD 계수)** | 무차원 (배수) | 0.5~20.0 | 4.0 / 17.5 | 기포 후보 판정 민감도. `임계값 = median + k × 1.4826 × MAD`. |

> **동작**: MAD(Median Absolute Deviation) 기반 통계적 이진화. DoG 결과에서 중앙값 + k×MAD 이상인 픽셀만 기포 후보가 된다.
> **낮은 k** = 민감(더 많은 후보, 오검출 증가), **높은 k** = 둔감(확실한 기포만 검출).
> 1.4826은 정규분포 가정 시 MAD→σ 변환 계수이다.

### 3.7 형태학 정리

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **Morph Close** | px (홀수 커널) | 1~31 | 5 | 기포 후보의 내부 구멍을 메운다. |
| **Morph Open** | px (홀수 커널) | 1~31 | 3 / 5 | 작은 점 노이즈를 제거한다. |

### 3.8 형상 필터

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **Min Diameter** | px | 1~500 | 8 / 10 | 최소 기포 직경. `minEnclosingCircle` 반지름이 이 값/2 미만이면 제거. |
| **Max Diameter** | px | 1~1,000 | 100 / 150 | 최대 기포 직경. 이보다 큰 원형 객체는 기포가 아닌 것으로 판단. |
| **Circularity Min** | 비율 | 0.00~1.00 | 0.35 / 0.4 | `4π × Area / Perimeter²`. 기포는 둥글므로 0.35~0.5 이상을 권장. |
| **Solidity Min** | 비율 | 0.00~1.00 | 0.70 | `Area / ConvexHullArea`. 기포는 꽉 찬 형태이므로 0.7 이상 권장. |
| **Max Aspect Ratio** | 무차원 | 1.0~10.0 | 2.2 | 장축/단축 비. 기포는 거의 원형이므로 2.0~2.5가 적합. |

> **면적 자동 계산**: Diameter로부터 내부적으로 면적 범위가 자동 계산된다.
> `min_area = π × (min_diameter/2)² × 0.35`, `max_area = π × (max_diameter/2)² × 1.30`
> 즉 **Diameter, Circularity, Solidity, Aspect Ratio 모두 AND 조건**으로 전부 통과해야 기포로 확정된다.

## 4. 탭 3: ECC 정렬 검사 (ECC Alignment Inspection)

기준 프레임(Reference)과 현재 프레임을 픽셀 단위로 정렬한 후, 차영상에서 이물을 검출한다.

### 4.1 전체 파이프라인 요약

```
프레임 입력 → 상태 게이트 판정 → [stop_confirmed일 때만]
→ 기준 프레임과 정렬(Alignment) → 유효 마스크 생성 → Safe Liquid ROI 적용
→ 차영상 계산 → Sweet Spot 판정 → 이진화 → Blob 필터 → [트래킹] → 최종 결과
```

### 4.2 기준 템플릿 관리

| 기능 | 설명 |
|------|------|
| **현재 화면 저장** | 현재 화면과 검사 ROI를 ECC 기준(Reference)으로 영구 저장한다. |
| **템플릿 초기화** | 저장된 기준 템플릿을 삭제하고 임시 기준으로 되돌린다. |

### 4.3 Global Motion Score

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **상한 (motion_threshold_high)** | gray level² (MSE) | 1.0~200.0 | 30.0 / 49.0 | 프레임 간 전역 MSE가 이 값 이상이면 액체 유동이 커서 검사를 건너뛴다. |
| **하한 (motion_threshold_low)** | gray level² (MSE) | 0.0~50.0 | 1.0 / 0.5 | 차영상 평균이 이 값 이하이면 거의 정지 상태로 low confidence 처리. |

> **연동**: 상한을 높이면 더 많은 움직임을 허용하여 검사가 덜 중단된다. 하한을 높이면 미세한 변화도 무시하여 정보량이 부족한 프레임에서의 오검출을 줄인다.

### 4.4 정렬 알고리즘

| 알고리즘 | 속도 | 회전 지원 | 특징 |
|----------|------|-----------|------|
| **ECC** | 느림 (해상도 의존) | O | 이미지 세기 기반 반복 최적화. 가장 정확하지만 큰 해상도에서 느릴 수 있다. |
| **Phase Correlate** | 매우 빠름 (30~50ms) | X (이동만) | FFT 기반 위상 상관. dx/dy만 계산하며 회전은 무시한다. |
| **ORB Feature** | 빠름 (10~30ms) | O | 특징점 매칭 + RANSAC. 회전 포함. 텍스처가 부족한 영상에서 실패 가능. |

#### ECC 정렬 세부 설정 (ECC 모드 선택 시에만 표시)

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **해상도 축소 (ecc_downscale_factor)** | 배율 | 1x / 2x / 4x / 8x | 2x / 4x | 정렬 연산 전 영상을 축소하여 속도를 높인다. 4x면 가로세로 1/4로 줄여 연산량이 약 1/16로 감소. 높을수록 빠르지만 정밀도가 미세하게 떨어진다. |
| **최대 반복 횟수 (ecc_max_iter)** | 회 | 10~5,000 | 200 / 100 | ECC 알고리즘의 반복 상한. 높이면 수렴 정밀도가 높지만 최악 시 처리 시간이 증가한다. |
| **수렴 기준 ε (ecc_epsilon)** | 무차원 (상관계수 변화) | 1e-7 ~ 0.1 | 1e-4 / 0.001 | 반복 간 상관계수 변화가 이 값 이하이면 수렴으로 판단하고 조기 종료한다. 작을수록 정밀하다. |
| **Gaussian 필터 (ecc_gauss_filt_size)** | px (홀수 커널) | 1~31 | 5 / 4 | 정렬 전 노이즈를 줄이는 가우시안 필터 크기. 정렬 안정성을 높인다. |

> **연동**: `downscale_factor`를 높이면 속도는 빨라지지만 `max_iter`와 `epsilon`의 수렴 효과가 줄어든다. 일반적으로 downscale 4x + max_iter 100 + epsilon 0.001이면 충분하다.

### 4.5 정렬 ROI

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **전체 프레임** | — | — | — | 전체 프레임을 정렬 영역으로 사용한다. 정확하지만 느리다. |
| **검사 ROI + 패딩** | — | — | — | 검사 ROI 주변만 정렬에 사용하여 속도를 높인다. |
| **좌우 패딩 (alignment_padding_x)** | px | 0~500 | 50 / 300 | 검사 ROI 좌우로 확장할 픽셀 수. 클수록 정렬 정보가 풍부해지지만 속도가 느려진다. |
| **상하 패딩 (alignment_padding_y)** | px | 0~500 | 50 / 300 | 검사 ROI 상하로 확장할 픽셀 수. |

> **연동**: 패딩 모드에서 패딩이 너무 작으면 정렬에 필요한 특징이 부족하여 실패할 수 있다. 검사 ROI 크기의 20~50%가 적절하다.

### 4.6 정렬·검사 영역

| 모드 | 설명 |
|------|------|
| **전체 프레임 정렬 후 ROI 검사** | 전체 프레임에 warp를 적용한 뒤 ROI 안에서만 blob 검사. 디버그 시 전체 정렬 영상 확인 가능. |
| **ROI 영역만 정렬·검사 (use_roi_only_warp)** | 검사 ROI만 warp·차영상·blob 연산. ROI가 작으면 속도가 크게 빨라진다. 결과는 동일하다. |

### 4.7 차영상 Sweet Spot

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **상한 (diff_too_high)** | gray level (차영상 평균) | 1.0~255.0 | 40.0 / 51.0 | 유효 마스크 영역의 차영상 평균이 이 값 이상이면 아직 유동이 크거나 정렬 실패로 판단하여 검사를 스킵한다. |
| **하한 (diff_too_low)** | gray level (차영상 평균) | 0.0~50.0 | 0.5 | 차영상 평균이 이 값 이하이면 변화가 거의 없어 low confidence로 처리한다. |

> **동작**: 정렬된 현재 프레임과 기준 프레임의 단방향 차이(`ref - current`, 즉 현재 프레임에서 새로 어두워진 부분만)를 구한 뒤, 유효 마스크 내부의 평균을 계산하여 판정한다.

### 4.8 Safe Liquid ROI

검사 ROI 내에서 유리벽, 액면, 바닥 등 노이즈가 많은 영역을 제외하고 **순수 액체 영역만** 검사 허용 구역으로 설정한다.

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **벽 제외 margin (safe_wall_margin)** | px | 0~500 | 20 | 검사 ROI 좌우 벽에서 안쪽으로 제외할 거리. 유리벽 굴절/반사 영역을 마스킹한다. |
| **상부/액면 band (safe_surface_band)** | px | 0~500 | 20 | 검사 ROI 상단 또는 추정된 액면으로부터 아래로 제외할 높이. 액면 출렁임을 마스킹한다. |
| **하부 제외 band (safe_bottom_band)** | px | 0~500 | 10 | 검사 ROI 하단에서 위로 제외할 높이. 바이알 바닥 굴절 영역 마스킹. |
| **최소 유효 면적 (safe_min_area)** | px² | 0~1,000,000 | 1,000 | 마스킹 후 남은 유효 면적이 이 값 미만이면 검사 불가(`invalid_safe_roi`)로 처리한다. |
| **액면 추정 사용 (use_surface_estimate)** | on/off | — | on | 상부 고정 band 외에 Sobel 기울기로 액면 위치를 자동 추정하여 제외 영역을 보수적으로 조정한다. |
| **액면 검색 비율 (surface_search_ratio)** | 비율 | 0.10~1.00 | 0.6 | 검사 ROI 상단에서 액면을 탐색할 높이 비율. 0.6이면 ROI 상단 60% 영역에서만 액면을 찾는다. |

> **연동**: `safe_wall_margin` + `safe_surface_band` + `safe_bottom_band` = 검사 ROI를 사방에서 축소시킨다. 너무 크게 설정하면 실제 검사 영역이 없어질 수 있으며, 이때 `safe_min_area` 검사에 걸려 검사가 스킵된다.

#### 표면 Dark 제외

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **상단 dark band 제외** | on/off | — | on | 액면 뒤쪽/상단부의 저휘도 띠를 추가로 제외한다. |
| **dark 임계값 (surface_dark_threshold)** | gray level | 0~255 | 30 | 이 밝기 이하인 픽셀을 dark band로 추출한다. |
| **dilate 커널 (surface_dark_dilate)** | px (홀수 커널) | 1~31 | 3 | 추출된 dark band를 확장(dilate)하여 제외 영역을 넓힌다. |

### 4.9 외곽 마스킹

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **외곽 마스킹 사용 (outer_mask_enabled)** | on/off | — | off | 정렬된 현재 영상에서 ROI 경계에 닿는 어두운 영역(유리벽)을 동적으로 제외한다. |
| **외곽 임계값 (outer_mask_threshold)** | gray level | 0~255 | 20 / 117 | 이 밝기 이하인 픽셀을 외곽 병 영역 후보로 추출. ROI 경계에 접촉하는 연결 성분만 실제 마스크가 된다. |

### 4.10 Blob 필터 (차영상 이물 검출)

| 파라미터 | 단위 | 범위 | 기본값 (코드/현재) | 동작 |
|----------|------|------|---------------------|------|
| **이진화 임계값 (blob_threshold)** | gray level | 1~255 | 25 / 30 | 차영상에서 이 값 이상 변화한 픽셀을 blob 후보로 추출한다. |
| **최소 면적 (blob_min_area)** | px² | 1~10,000 | 10 / 1 | blob 윤곽선 면적이 이 값 미만이면 제거. |
| **최대 면적 (blob_max_area)** | px² | 1~100,000 | 5,000 / 500 | blob 면적이 이 값 초과면 제거. |
| **최소 원형도 (blob_min_circularity)** | 비율 | 0.00~1.00 | 0.0 | 0이면 형상 제한 없음. |
| **Morph Open** | px (홀수 커널) | 1~31 | 3 | 이진화 후 작은 노이즈 제거. |
| **Morph Close** | px (홀수 커널) | 1~31 | 5 / 3 | 끊어진 blob 연결. |

> **연동**: `blob_threshold`가 낮으면 미세한 차이도 후보가 되어 검출 수가 많아지지만 오검출도 증가한다. `blob_min_area`와 함께 조절하여 적절한 민감도를 찾는다.

### 4.11 다프레임 통합 (Temporal Tracking)

단일 프레임의 일시적 노이즈를 배제하기 위해, **여러 프레임에서 반복 관찰된** 이물만 최종 불량으로 판정하는 추적 시스템이다.

| 파라미터 | 단위 | 범위 | 기본값 | 동작 |
|----------|------|------|--------|------|
| **연속 프레임 후보 통합 (track_enabled)** | on/off | — | on | 활성화 시 단일 프레임 blob를 바로 defect로 처리하지 않고, 반복 관찰된 후보만 유지한다. |
| **기록 프레임 수 (track_history_frames)** | 프레임 | 1~100 | 6 | 후보 track을 유지할 최대 프레임 수. 이 프레임 수 동안 재관찰되지 않으면 track이 소멸한다. |
| **연결 허용 거리 (track_match_distance)** | px (유클리드 거리) | 0.0~500.0 | 18.0 | 서로 다른 프레임의 후보를 같은 track으로 연결할 최대 중심점 거리. 이물이 미세하게 움직여도 동일 이물로 인식하는 범위. |
| **최소 재검출 횟수 (track_min_hits)** | 회 | 1~100 | 2 | 이 횟수 이상 연속 검출되어야 최종 defect로 확정. 1이면 트래킹 효과 없음(단일 프레임 즉시 확정). |
| **면적 변화 허용비 (track_max_area_ratio)** | 무차원 (배수) | 1.0~20.0 | 3.0 | 같은 track으로 연결할 때 `max(면적1, 면적2) / min(면적1, 면적2)`가 이 값 이내여야 한다. 이물 면적이 프레임마다 급변하면 별개 track으로 분리된다. |

> **연동**: `track_min_hits`를 높이면 오검출이 줄어들지만, 짧은 정지 구간에서 실제 이물을 놓칠 수 있다. `track_history_frames`와 `track_min_hits`의 관계: history 내에 min_hits 이상 관찰되어야 하므로, `min_hits ≤ history_frames`이어야 의미가 있다.
